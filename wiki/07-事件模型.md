# 事件模型

> 三阶段事件系统详解

---

## 概述

事件模型采用三阶段设计，每个高层行为都可分为准备、执行、结果三个阶段。

---

## 三阶段模型

### 阶段划分

| 阶段 | 前缀 | 说明 | 可修改性 |
|------|------|------|----------|
| BeforeX | Before | 准备/修改阶段，可修改输入参数 | 可写 |
| OnX | On | 执行阶段，只读执行 | 只读 |
| AfterX | After | 结果/连锁阶段，只读结果 | 只读 |

### 示例：攻击事件

```
BeforeAttack → 修改目标、伤害、是否取消
     ↓
OnAttack → 确认攻击发生
     ↓
AfterAttack → 触发连锁、附加效果
```

### 示例：受伤事件

```
BeforeDamaged → 修改伤害值、是否免疫
     ↓
OnDamaged → 确认受伤发生
     ↓
AfterDamaged → 触发连锁、附加效果
```

## 事件上下文结构

### EventData 结构

```json
{
  "core": {
    "damage": 10,
    "source": "entity_id",
    "target": "entity_id",
    "canceled": false,
    "tags": ["projectile"]
  },
  "runtime": {
    "event_id": "uuid",
    "depth": 1,
    "timestamp": 123456
  },
  "mods": {
    "modA": {
      "stack": 3
    }
  }
}
```

### 字段说明

#### core（核心数据）

| 字段 | 类型 | 说明 |
|------|------|------|
| `damage` | int | 伤害值 |
| `source` | string | 来源实体ID |
| `target` | string | 目标实体ID |
| `canceled` | bool | 是否取消（Before阶段可修改） |
| `tags` | array | 标签列表 |

#### runtime（运行时数据）

| 字段 | 类型 | 说明 |
|------|------|------|
| `event_id` | string | 事件唯一ID |
| `depth` | int | 事件链深度 |
| `timestamp` | float | 时间戳 |

#### mods（MOD数据）

| 字段 | 类型 | 说明 |
|------|------|------|
| `modId` | object | MOD自定义数据 |

## 事件广播

### 核心层广播

```csharp
class CoreLayer {
    private EventManager _eventManager;

    public void DamageEntity(Entity source, Entity target, int damage) {
        // Before阶段：允许修改
        var beforeData = new EventData {
            core = new Dictionary<string, object> {
                ["damage"] = damage,
                ["source"] = source.id,
                ["target"] = target.id,
                ["canceled"] = false
            },
            runtime = new Dictionary<string, object> {
                ["event_id"] = Guid.NewGuid().ToString(),
                ["depth"] = 1,
                ["timestamp"] = Time.time
            }
        };

        _eventManager.Broadcast($"{target.type}.before_damaged", beforeData);

        // 检查是否取消
        if ((bool)beforeData.core["canceled"]) {
            return;
        }

        // 执行伤害
        int finalDamage = (int)beforeData.core["damage"];
        target.TakeDamage(finalDamage);

        // On阶段：只读
        var onData = new EventData(beforeData);
        _eventManager.Broadcast($"{target.type}.damaged", onData);

        // After阶段：只读
        var afterData = new EventData(onData);
        _eventManager.Broadcast($"{target.type}.after_damaged", afterData);
    }
}
```

## 事件订阅

### 触发器订阅

```csharp
class TriggerInstance {
    public void Subscribe() {
        EventManager.Subscribe(event_name, OnEvent);
    }

    public void Unsubscribe() {
        EventManager.Unsubscribe(event_name, OnEvent);
    }

    private void OnEvent(EventData eventData) {
        if (!is_enabled) return;

        var strategy = TriggerStrategyRegistry.Get(def_id);
        if (strategy.CheckCondition(eventData, condition_values, plantState)) {
            // 执行效果树
            foreach (var effectTree in bound_effects) {
                EffectExecutor.Execute(effectTree, eventData);
            }

            // 更新冷却
            last_triggered_time = Time.time;
        }
    }
}
```

## 事件链

### 事件链形成

```mermaid
graph LR
    A[事件1触发] --> B[执行效果树]
    B --> C[子效果广播事件2]
    C --> D[执行效果树]
    D --> E[子效果广播事件3]
    E --> F[执行效果树]
    F --> G[事件链自然终止]
```

### 事件链终止条件

1. **深度超限**：连锁深度 > 5，事件静默丢弃
2. **无子效果**：效果树遍历完成，没有新事件广播
3. **条件不满足**：触发器条件检查失败
4. **策略终止**：效果策略返回 `terminated = true`
5. **事件取消**：Before阶段设置 `canceled = true`

### 事件链示例

```
plant.before_damaged (受伤前)
  ↓ (可修改伤害、取消)
plant.damaged (受伤)
  ↓
on_hit效果执行
  ↓
projectile.hit (子弹命中)
  ↓
explode效果执行
  ↓
zombie.before_died (死亡前)
  ↓ (可取消死亡)
zombie.died (死亡)
  ↓
summon效果执行
  ↓
事件链终止（深度=5）
```

## 事件优先级

### 优先级定义

| 优先级 | 说明 |
|--------|------|
| 0 | 最高优先级，系统核心事件 |
| 1 | 高优先级，关键游戏逻辑 |
| 2 | 中优先级，普通游戏逻辑 |
| 3 | 低优先级，视觉效果、音效 |

### 优先级使用

```csharp
class EventManager {
    public void Subscribe(string eventName, EventHandler handler, int priority = 2) {
        var list = GetSubscriberList(eventName);
        list.Add((handler, priority));
        list.Sort((a, b) => a.priority.CompareTo(b.priority));
    }
}
```

### 示例

```csharp
// 系统核心事件，最高优先级
EventManager.Subscribe("plant.damaged", SystemDamageHandler, priority: 0);

// MOD事件，低优先级
EventManager.Subscribe("plant.damaged", ModDamageHandler, priority: 3);
```

## 事件过滤

### 标签过滤

```csharp
class TriggerInstance {
    private List<string> _requiredTags;
    private List<string> _excludedTags;

    public bool ShouldHandle(EventData eventData) {
        var tags = (List<string>)eventData.core["tags"];

        // 检查必需标签
        foreach (var tag in _requiredTags) {
            if (!tags.Contains(tag)) {
                return false;
            }
        }

        // 检查排除标签
        foreach (var tag in _excludedTags) {
            if (tags.Contains(tag)) {
                return false;
            }
        }

        return true;
    }
}
```

### 目标过滤

```csharp
class TriggerInstance {
    private string _targetType;

    public bool ShouldHandle(EventData eventData) {
        if (string.IsNullOrEmpty(_targetType)) {
            return true;
        }

        var target = EntityManager.Get((string)eventData.core["target"]);
        return target.type == _targetType;
    }
}
```

### 事件追踪

```csharp
class EventDebugger {
    private static Dictionary<string, List<EventData>> _eventHistory = new();

    public static void RecordEvent(string eventName, EventData eventData) {
        if (!_eventHistory.ContainsKey(eventName)) {
            _eventHistory[eventName] = new List<EventData>();
        }
        _eventHistory[eventName].Add(eventData);
    }

    public static List<EventData> GetEventHistory(string eventName) {
        return _eventHistory.ContainsKey(eventName) ? _eventHistory[eventName] : new List<EventData>();
    }
}
```
