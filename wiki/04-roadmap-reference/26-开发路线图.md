## 当前路线判断

> 第一到第六阶段已完成主干沉淀。项目已通过 ADR-001 从"第七阶段输入准备"正式切换到 **Mechanic-first 重构阶段**。重构三个阶段已完成。

这意味着路线图的起点不再是：

- 围绕 EntityTemplate / TriggerBinding 继续加厚内容
- 证明扩展包是否可行

而是：

- 在 Archetype + Mechanic[] 新作者模型上继续扩展
- 将旧内容迁移到 archetype 独立实例化
- 保持内容、验证和展示入口同步增长

决策依据见 [ADR-001 路线切换与重构范围](../decisions/ADR-001-路线切换与重构范围.md)。

## 已完成的重构阶段

### 重构第一阶段：编译链最小闭环

- `CombatArchetype` / `CombatMechanic` 10 个一级 family 冻结
- `MechanicCompiler` / `NormalizedMechanicSet` / `RuntimeSpec` 编译链
- `MechanicFamilyRegistry` / `MechanicTypeRegistry` / `MechanicCompilerRegistry`
- `EntityFactory` 双路径（archetype 优先 + 旧 template 兜底）
- 5 个 archetype 骨架验证通过

### 重构第二阶段：Controller / State / Lifecycle 最小路径

- `ControllerComponent` + `ControllerRegistry`（core.bite / core.sweep）
- `StateComponent`（时间触发 + 事件触发 transition）
- `Lifecycle.on_spawned` / `on_place` 编译路径

### 重构第三阶段：编译链补完与扩展

- Multi-payload 编译（一个 trigger 驱动多个 payload）
- Per-type compiler dispatch（`MechanicCompilerRegistry` 可分发编译 callable）
- State 扩展：core.growth / core.rage
- Lifecycle 扩展：core.on_armed / core.on_state_enter / core.on_expire / core.on_removed
- Controller 策略增强：DetectionRegistry 集成 + blackboard
- 确定性随机协议：battle_seed + seed 派生 + ShuffleBag
- Mechanic-local runtime state
- Archetype 独立实例化（无 backend_entity_template 路径）
- 迁移对照验证（sunflower / peashooter / zombie parity）

---

## 当前进行中的路线段

### 1. 扩展已编译 family 的 type 覆盖面

当前已编译的 family 为 Trigger / Lifecycle / Payload / Controller / State / Targeting / Emission / Trajectory / HitPolicy。Placement 需要建立编译路径。

### 2. 将更多旧内容迁移到 archetype

当前 50 个 archetype 中部分仍依赖 `backend_entity_template`。继续将正式内容迁移为独立 archetype。

### 3. 保持回归验证同步

新增机制或 archetype 时同步补验证场景、更新 `tools/validation_scenarios.json`、更新验证矩阵。

---

## 当前阶段的退出信号

Mechanic-first 重构的退出信号：

- 所有 10 个 family 都有编译路径覆盖
- 全部正式内容已迁移到独立 archetype（不再依赖 backend_entity_template）
- 旧 EntityTemplate / TriggerBinding 路径可安全废弃
- 迁移对照验证全部通过

---

## 后续阶段应该怎么理解

### 方向 A：继续加厚战斗内容

完成重构退出后，优先扩展正式内容厚度。

### 方向 B：开始外围组织层的最小接入

只有当战斗内容基线足够厚时才进入。

### 方向 C：加强作者工作流与轻量工具

逐步进入，服务于当前内容生产和验证。

---

## 当前不应优先的方向

- 完整 ECS 化
- 大型编辑器或 DSL
- 完整社区工坊
- 为了"像成品"而过早做大规模 UI 包装
- 让外围产品壳倒逼当前战斗主干结构
