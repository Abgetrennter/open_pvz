## 注册表详解

### CharacterRegistry

> 文件: `scripts/autoload/global/character_registry.gd`

包含的枚举：

| 枚举 | 用途 |
|------|------|
| `CharacterType` | 区分植物阵营 / 僵尸阵营 |
| `PlantType` | 每种植物对应一个枚举值（001~048） |
| `ZombieType` | 每种僵尸对应一个枚举值（001~026） |
| `ZombieRowType` | 僵尸行类型（地面/水中/水中底部） |

用途：
- 角色的 `plant_type` / `zombie_type` 导出属性引用此枚举
- 场景注册表按枚举值索引对应 PackedScene
- 管理器按枚举类型分发行为

### BulletRegistry

> 文件: `scripts/autoload/global/bullet_registry.gd`

| 枚举 | 用途 |
|------|------|
| `BulletType` | 每种子弹对应一个枚举值 |

用途：
- 子弹的 `bullet_type` 导出属性引用此枚举
- 攻击组件根据类型创建对应子弹

### ItemRegistry

> 文件: `scripts/autoload/global/item_registry.gd`

| 枚举 | 用途 |
|------|------|
| `ItemType` | 游戏物品类型枚举 |

### MainSceneRegistry

> 文件: `scripts/autoload/global/main_scene_registry.gd`

包含主场景的路径常量，用于场景切换时的路径引用。

## 场景注册与实例化流程

```
1. SceneRegistry (autoload) 在启动时预加载所有 PackedScene
2. 代码通过 SceneRegistry 的属性获取 PackedScene 引用
3. 调用 .instantiate() 创建实例
4. 设置导出属性 (type, camp, position 等)
5. add_child() 加入场景树

关键文件:
  scripts/autoload/scene_registry.gd   — PackedScene 仓库
  scripts/autoload/global/character_registry.gd — 类型枚举定义
  scripts/autoload/global/bullet_registry.gd    — 子弹类型枚举
```

## 具体植物实现模式

以豌豆射手为例：

```
plant_001_pea_shooter_single.gd
  extends Plant000Base
  ├── 导出: 覆写 plant_type = CharacterRegistry.PlantType.PEA_SHOOTER_SINGLE
  ├── 组件: ComponentAttackBulletBase (攻击)
  ├── 组件: HpComponentPlant (血量)
  ├── 组件: ComponentAnim (动画)
  └── 组件: CreateSunComponent (无) / BlinkComponent (无)
```

每个具体植物的脚本通常只做：
1. 设置 plant_type 枚举
2. 在场景中组合需要的组件
3. 覆写少量特殊行为

大部分差异在场景文件 (.tscn) 的组件组合方式上体现，而非脚本代码。
