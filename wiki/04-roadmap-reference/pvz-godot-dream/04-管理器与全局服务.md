# 管理器与全局服务

> 本页索引所有管理器和 Autoload 全局服务，列出核心方法便于快速定位。

## Autoload 全局服务

### Global（全局门面）

> 文件: `scripts/autoload/global/global.gd`

门面模式，代理访问以下子服务：

| 子服务 | 脚本路径 | 职责 |
|-------|---------|------|
| CharacterRegistry | `scripts/autoload/global/character_registry.gd` | 角色/植物/僵尸类型枚举 |
| BulletRegistry | `scripts/autoload/global/bullet_registry.gd` | 子弹类型枚举 |
| ItemRegistry | `scripts/autoload/global/item_registry.gd` | 物品类型枚举 |
| MainSceneRegistry | `scripts/autoload/global/main_scene_registry.gd` | 主场景路径常量 |
| ConfigService | `scripts/autoload/global/config_service.gd` | 配置管理 |
| GlobalGameState | `scripts/autoload/global/global_game_state.gd` | 全局游戏状态 |
| GlobalReadData | `scripts/autoload/global/global_read_data.gd` | 数据读取服务 |
| SaveService | `scripts/autoload/global/save_service.gd` | 存档/读档 |
| UserManager | `scripts/autoload/global/user_manager.gd` | 用户管理 |

### EventBus（事件总线）

> 文件: `scripts/autoload/event_bus.gd`
> class_name: EventBus

| 特性 | 说明 |
|------|------|
| 动态事件名 | 字符串事件名，无需预定义信号 |
| 订阅 | `subscribe(event_name, callback, priority, filter)` |
| 退订 | `unsubscribe(event_name, callback)` |
| 一次性订阅 | `subscribe_once(event_name, callback)` |
| 优先级 | 数值越小越先执行 |
| 过滤器 | 订阅时指定过滤函数 |
| 历史记录 | `enable_history` + `get_event_history()` |
| 调试模式 | `debug_mode` 打印事件日志 |
| 信号 | `event_pushed`, `event_handled` |

### SceneRegistry（场景注册表）

> 文件: `scripts/autoload/scene_registry.gd`
> class_name: SceneRegistry

启动时预加载所有游戏 PackedScene，通过属性直接获取引用。

### SoundManager（音频管理）

> 文件: `scenes/autoload/sound_manager.tscn`（场景自带脚本）

管理背景音乐和音效播放。

### TreePauseManager（暂停管理）

> 文件: `scripts/autoload/tree_pause_manager.gd`

基于暂停因子的暂停系统，支持多个暂停源叠加。

### AllCards（卡牌系统）

> 文件: `scripts/autoload/global/all_cards.gd`

管理所有可用卡牌的注册和查询。

### GlobalUtils（全局工具）

> 文件: `scripts/autoload/util/global_utils.gd`
> class_name: GlobalUtilsClass

| 方法 | 用途 |
|------|------|
| update_plant_cell_slope_y() | 更新斜坡格子 Y 坐标 |
| format_number_with_commas() | 数字格式化 |
| get_dic_product() | 字典乘积 |
| get_character_type() | 判断角色类型（植物/僵尸） |
| create_new_timer_once() | 创建一次性计时器 |
| get_curr_time() | 获取当前时间 |

### RandomPicker（加权随机）

> 文件: `scripts/autoload/util/random_picker.gd`
> class_name: RandomPicker

加权随机选择工具。

## 管理器体系

> 所有管理器在主战斗场景中由 MainGameManager 创建和协调。

### MainGameManager（主战斗控制器）

> 文件: `scripts/manager/main_game_manager.gd`
> class_name: MainGameManager

| 导出属性 | 说明 |
|---------|------|
| test_change_sun_value | 测试用：直接设置阳光值 |
| test_death_all_zombie | 测试用：杀死所有僵尸 |
| test_time_scale | 游戏速度倍率 |
| main_game_progress | 游戏进度状态 (E_MainGameProgress) |
| game_para: ResourceLevelData | 关卡参数资源 |
| is_test | 是否测试模式 |

| 核心方法 | 用途 |
|---------|------|
| set_game_para() | 设置关卡参数 |
| main_game_start() | 启动战斗 |
| win_main_game() | 胜利处理 |
| save_game_main_game() | 保存战斗状态 |
| load_game_main_game() | 加载战斗状态 |
| start_next_round_game() | 开始下一波次 |

### MainGameSubManager（子管理器协调）

> 文件: `scripts/manager/main_game_sub_manager.gd`

协调各子管理器之间的交互。

## 管理器间通信方式

```
1. 直接引用: MainGameManager 持有所有子管理器的引用
2. EventBus: 跨系统通信（如僵尸死亡通知、阳光变化）
3. 信号: 组件向管理器上报（如角色死亡信号）
4. Autoload: 通过全局单例访问共享状态
```
