# PVZ-Godot-Dream 架构总览

- 状态：当前事实


> 本页描述 `vendor/PVZ-Godot-Dream` 的整体架构模式、系统连接关系和启动流程。

---

## 设计模式总览

本项目采用以下核心模式：

| 模式 | 实现位置 | 说明 |
|------|---------|------|
| **Autoload 单例** | `scripts/autoload/` + `project.godot` | 全局服务通过 Godot Autoload 机制注册 |
| **事件总线** | `scripts/autoload/event_bus.gd` | 解耦的系统间通信，支持优先级和过滤 |
| **组件化角色** | `scripts/character/components/` | 角色本体持有基础属性，行为由组件实现 |
| **注册表** | `scripts/autoload/global/*_registry.gd` | 集中管理类型枚举，用于标识和查找 |
| **场景注册** | `scripts/autoload/scene_registry.gd` | 所有 PackedScene 的预加载中心 |
| **管理器集群** | `scripts/manager/` | 主战斗场景拆分为多个子管理器 |
| **继承体系** | `character_000_base` → `plant_000_base` / `zombie_000_base` | 三层角色继承 |

## 系统连接图

```
project.godot (Autoload 注册)
│
├─→ Global (门面)
│     ├─→ CharacterRegistry   (类型枚举)
│     ├─→ BulletRegistry      (类型枚举)
│     ├─→ ItemRegistry        (类型枚举)
│     ├─→ MainSceneRegistry   (场景路径)
│     ├─→ ConfigService       (配置)
│     ├─→ GlobalGameState     (游戏状态)
│     ├─→ GlobalReadData      (数据读取)
│     ├─→ SaveService         (存档)
│     └─→ UserManager         (用户)
│
├─→ EventBus ──── 全局事件通信
│
├─→ SceneRegistry ── PackedScene 预加载仓库
│
├─→ SoundManager ── 音频管理
│
├─→ AllCards ────── 卡牌系统
│
├─→ GlobalUtils ─── 工具函数
│
└─→ TreePauseManager ── 暂停因子
```

## 启动流程

```
1. Godot 加载 project.godot
2. Autoload 节点按注册顺序初始化:
   Global → SoundManager → SceneRegistry → AllCards → GlobalUtils → EventBus → TreePauseManager
3. 主场景加载 (MainGame 或 01StartMenu)
4. 用户进入战斗 → MainGameManager 初始化
5. MainGameManager 创建子管理器:
   ├── CardManager (卡牌)
   ├── ZombieManager (僵尸波次)
   ├── PlantCellManager (格子)
   ├── DropItemManager (掉落)
   ├── GameItemManager (物品)
   ├── HandManager (手持)
   └── BackgroundManager (背景)
6. 子管理器通过 EventBus 和直接引用协调工作
```

## 角色生命周期

```
注册表查询类型 (CharacterRegistry.PlantType / ZombieType)
       │
       ▼
SceneRegistry 获取 PackedScene
       │
       ▼
instantiate() 创建角色节点
       │
       ▼
init_plant() / init_zombie()  ── 设置类型、位置、状态
       │
       ▼
ready_norm()  ── 战斗模式初始化
  ├── 加载 HP 组件
  ├── 加载攻击组件
  ├── 加载动画组件
  └── 注册到管理器
       │
       ▼
游戏中运行 (组件驱动行为)
       │
       ▼
character_death()  ── 死亡处理
  ├── 播放死亡动画
  ├── 掉落物品
  ├── 通知管理器
  └── queue_free()
```

## 子弹生命周期

```
攻击组件触发发射
       │
       ▼
BulletRegistry.BulletType 确定类型
       │
       ▼
SceneRegistry 获取子弹 PackedScene
       │
       ▼
instantiate() + 设置阵营/类型
       │
       ▼
挂载移动组件:
  ├── Linear   ── 直线飞行
  ├── Parabola ── 抛物线
  └── Track    ── 追踪目标
       │
       ▼
每帧移动 + 碰撞检测
       │
       ▼
命中目标 → 造成伤害 → 特效 → 销毁
```

## 目录设计哲学

| 目录 | 放什么 | 不放什么 |
|-----|--------|---------|
| `scenes/` | .tscn 场景文件，按功能分子目录 | 脚本和纯逻辑 |
| `scripts/` | .gd 脚本，镜像 scenes/ 的目录结构 | 资源文件 |
| `data/` | JSON/TRES 纯数据 | 脚本 |
| `resources/` | 运行时资源（样式、角色状态资源等） | 场景 |
| `animation/` | Reanim 导出的动画资源 | 脚本 |
| `shaders/` | .gdshader 和 .gdshaderinc | 其他 |

scenes/ 和 scripts/ 的子目录结构基本镜像对应，便于查找：
- `scenes/character/plant/` ↔ `scripts/character/plant/`
- `scenes/bullet/` ↔ `scripts/bullet/`
- `scenes/main/` ↔ `scripts/manager/`（不完全对应，管理器脚本统一在 manager/ 下）

## 命名约定

| 对象 | 命名规则 | 示例 |
|------|---------|------|
| 角色基类 | `NNN_类别_base` | `character_000_base`, `plant_000_base` |
| 具体角色 | `NNN_英文名` | `plant_001_pea_shooter_single`, `zombie_003_cone` |
| 组件 | `component_功能` 或 `类别Component` | `ComponentAttackBulletBase`, `HpComponent` |
| 子弹 | `bullet_NNN_名称` | `bullet_002_pea_snow`, `bullet_012_melon` |
| 管理器 | `XxxManager` | `ZombieManager`, `DropItemManager` |
| 注册表 | `XxxRegistry` | `CharacterRegistry`, `BulletRegistry` |
| 场景 | 与脚本同名 | `plant_001_pea_shooter_single.tscn` |
| 主场景 | 数字前缀 + 功能名 | `01StartMenu.tscn`, `02AdventureChooesLevel.tscn` |
