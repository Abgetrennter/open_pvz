---

## 一句话结论

素材包是扩展系统中的正式包类型，负责把逻辑表现 ID 解析为真实视觉、音频和主题资源。结合原版 `.reanim` 迁移现实，素材包不只是贴图目录，还要能承载 Godot actor scene、VisualProfile、Action Recipe、Part Slot、VisualFx 和 AudioCue。

原版素材包必须支持本地私有发布策略：可在开发者本机运行，但默认不提交、不发布、不进入主仓 CI 发布物。

---

## 设计目标

素材包系统要同时满足四个目标：

1. 主仓无原版素材也能运行。
2. 内容包不直接绑定真实资源路径。
3. 同一内容可以切换不同素材主题。
4. 原版资源和原版派生产物可以在本地验证，但不能误发布。

---

## 素材包负责什么

`asset_pack` 可以提供：

- 逻辑表现 ID 到资源的映射
- `VisualProfileDef`
- actor scene / composite actor scene
- Action Recipe / Part Slot 配置
- `VisualFxDef`
- `AudioCueDef`
- 贴图、音效、UI 图标、主题资源
- 导入报告和生成元数据

`asset_pack` 不负责：

- `CombatArchetype` 正文
- 战斗数值
- 目标查询、伤害、冷却、命中等规则
- 关卡或波次正文

---

## 逻辑表现 ID

内容包应引用逻辑表现 ID，而不是具体资源路径。

建议命名：

```text
entity.plant.peashooter.visual
entity.plant.sunflower.visual
projectile.pea.visual
card.plant.peashooter.icon
vfx.jalapeno.lane_fire
sfx.projectile.pea.hit
ui.icon.sun
```

素材包负责把这些 ID 解析到具体资源。

当前 `AssetRegistry` 尚未实现，因此过渡期可以继续通过 `VisualProfileRegistry` 注册具体 `VisualProfileDef`，但 ID 设计应提前按逻辑表现 ID 收口。

---

## 当前代码可用路径

当前已经可以正规接入的视觉资源 slot：

| register kind | Registry | 资源目录 |
|---------------|----------|----------|
| `visual_profiles` | `VisualProfileRegistry` | `data/combat/visual_profiles` |
| `visual_cues` | `VisualCueRegistry` | `data/combat/visual_cues` |
| `visual_fx` | `VisualFxRegistry` | `data/combat/visual_fx` |
| `audio_cues` | `AudioCueRegistry` | `data/combat/audio_cues` |

这意味着第一版素材包不必等待完整 `AssetRegistry`，可以先作为 data-only 视觉资源包接入。

示例当前兼容 manifest：

```json
{
  "pack_id": "classic_original_assets",
  "enabled_by_default": false,
  "trust_level": "data_only",
  "register": ["visual_profiles", "visual_cues", "visual_fx", "audio_cues"],
  "capabilities": ["visual_profiles", "visual_cues", "visual_fx", "audio_cues"]
}
```

## 本地私有包根目录

正式设计新增本地私有扩展根：

```text
res://local_extensions
```

用途：

- 存放本机私有素材包。
- 存放原版素材导出产物。
- 存放依赖私有素材的 collection pack。

约束：

- `local_extensions/` 必须进入 `.gitignore`。
- 默认不作为发布物。
- 加载器应支持扫描该目录，但可通过 manifest 或 CLI 控制启用。
- CI 默认不依赖该目录存在。

当前代码尚未扫描 `res://local_extensions`；落地时应在 `ExtensionPackCatalog._list_scan_roots()` 中增加该根，并补 guardrail 验证。

## 原版迁移包拆分

推荐结构：

```text
extensions/
  openpvz_classic_semantics/
  openpvz_classic_content/
  openpvz_placeholder_assets/

local_extensions/
  classic_original_assets/
  classic_original_collection/
```

### openpvz_classic_semantics

公开发布。

内容：

- `.reanim` 导入 manifest 模板
- Action Recipe / Part Slot 语义模板
- 迁移文档
- 不含原版素材和派生产物

### openpvz_classic_content

可公开发布。

内容：

- 原版风格玩法内容定义
- 只引用逻辑表现 ID
- 不引用 `vendor/out_files` 或本地素材路径

### openpvz_placeholder_assets

公开发布。

内容：

- 合法占位 actor
- 占位卡片图标
- 占位 vfx/audio
- 作为无私有素材时的回退

### classic_original_assets

本地私有，不发布。

内容：

- 原版 `.reanim/.png/.ogg` 的导出产物
- `actor.tscn`
- `visual_profile.tres`
- `visual_fx.tres`
- `audio_cue.tres`
- `import_report.json`

### classic_original_collection

本地私有，不发布。

内容：

- 组合 `openpvz_classic_content` 与 `classic_original_assets`
- 提供本机“原版风格体验”入口

## Action Recipe 在素材包中的位置

当前 Action Recipe 仍处于迁移验证阶段，保存在 `tools/reanim_importer/reanim_visual_manifest.json` 中。

正式路线：

1. 过渡期：Action Recipe 继续作为本地导入 manifest 字段。
2. 稳定后：抽出 `VisualActionRecipeDef`。
3. 再稳定后：新增 `visual_action_recipes` register kind 和 registry。
4. 最终 asset_pack 通过 `visual_profiles + visual_action_recipes + visual_fx + audio_cues` 组合完整表现。

第一版不应设计完整时间轴语言，只收敛少量必要 op：

- `play_state`
- `play_action`
- `part_action`
- `root_motion`
- `emit_local_cue`
- `spawn_world_fx`
- `return_home`
- `hide_actor`

---

## 发布守卫

正式发布前必须满足：

- 公开包不得包含原版 `.reanim/.png/.ogg`。
- 公开包不得包含由原版素材生成的 actor、贴图、音频等派生产物。
- 公开包不得引用 `vendor/out_files`。
- 公开包不得引用 `local_extensions`。
- CI 不应依赖 `local_extensions` 存在。
- 本地私有包必须通过 `.gitignore` 和 manifest `publish_policy = local_private` 双重标记。

---

## 最小落地顺序

1. 文档正式化：固定包边界、本地私有根、发布守卫。
2. 增加 `local_extensions/` ignore 与扫描根。
3. 建立一个本地 `classic_original_assets` 包，把当前 `vendor/out_files/_openpvz_import` 生成物迁入。
4. 公开 `openpvz_classic_semantics`，只放导入 manifest 模板和语义配置。
5. 为缺失私有素材的 showcase/validation 提供跳过或提示机制。
6. 再实现 `AssetRegistry` 与真正的 `asset_index`。
