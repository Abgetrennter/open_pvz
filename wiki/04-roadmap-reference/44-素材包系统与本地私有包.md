---

## 一句话结论

素材包是扩展系统中的正式包类型，负责把逻辑表现 ID 解析为真实视觉、音频和主题资源。结合原版 `.reanim` 迁移现实，素材包不只是贴图目录，还要能承载 Godot actor scene、VisualProfile、Action Recipe、Part Slot、VisualFx 和 AudioCue。

原版素材包必须支持本地私有发布策略：可在开发者本机运行，但默认不提交、不发布、不进入主仓 CI 发布物。

---

## 设计目标

素材包系统要同时满足四个目标：

1. 主仓无原版素材也能运行。
2. 内容包不直接绑定真实资源路径。
3. 同一内容可以切换不同素材主题。
4. 原版资源和原版派生产物可以在本地验证，但不能误发布。

---

## 素材包负责什么

`asset_pack` 可以提供：

- 逻辑表现 ID 到资源的映射
- `VisualProfileDef`
- actor scene / composite actor scene
- Action Recipe / Part Slot 配置
- `VisualFxDef`
- `AudioCueDef`
- 贴图、音效、UI 图标、主题资源
- 导入报告和生成元数据

`asset_pack` 不负责：

- `CombatArchetype` 正文
- 战斗数值
- 目标查询、伤害、冷却、命中等规则
- 关卡或波次正文

---

## 逻辑表现 ID

内容包应引用逻辑表现 ID，而不是具体资源路径。

建议命名：

```text
entity.plant.peashooter.visual
entity.plant.sunflower.visual
projectile.pea.visual
card.plant.peashooter.icon
vfx.jalapeno.lane_fire
sfx.projectile.pea.hit
ui.icon.sun
```

素材包负责把这些 ID 解析到具体资源。

当前最小 `AssetRegistry` 已支持 `visual_profile` 解析：优先从已启用素材包的 `asset_index.json` 读取逻辑表现 ID，再加载对应 `VisualProfileDef`。`VisualProfileRegistry` 继续作为现有 slot 注册表和回退路径。

---

## 运行时资产格式 v1

原版素材包保留两层资产：

- `sources/`：原始私有输入，包括 `.reanim`、png、`resources.xml`、ogg 等，只用于导入和复现。
- 运行时层：`actors/`、`data/combat/visual_profiles/`、`data/combat/visual_fx/`、`data/combat/audio_cues/`，这是 OpenPVZ 运行时实际消费的 Godot 资源。

`asset_index.json` 是这两层之间的索引。`AssetIndexCatalog` 负责读取和校验索引；`AssetRegistry` 是运行时入口，负责把逻辑表现 ID 解析成 `VisualProfileDef`。第一版只支持 `visual_profile`，不处理主题覆盖、FX、音频或包优先级。

当前 v1 结构：

```json
{
  "version": 1,
  "format": "openpvz.asset_index.v1",
  "pack_id": "classic_original_assets",
  "assets": {
    "classic_original.entity.plant.peashooter.visual": {
      "kind": "visual_profile",
      "path": "data/combat/visual_profiles/plants/peashooter.tres",
      "actor_scene": "actors/peashooter/actor.tscn",
      "source": {
        "reanim": "sources/reanim/PeaShooterSingle.reanim",
        "resources": "sources/properties/resources.xml"
      },
      "generated": {
        "raw_actor_scene": "generated/raw/peashooter/actor.tscn",
        "import_report": "generated/reports/peashooter.import_report.json",
        "composite_report": "generated/reports/peashooter.composite_report.json"
      }
    }
  },
  "visual_profiles": {
    "classic_original.entity.plant.peashooter.visual": "data/combat/visual_profiles/plants/peashooter.tres"
  }
}
```

`assets` 是正式结构；`visual_profiles` 是过渡期兼容字段，方便现有 `VisualProfileRegistry` 扫描路径与 `AssetRegistry` 解析路径共存。

## 当前代码可用路径

当前已经可以正规接入的视觉资源 slot：

| register kind | Registry | 资源目录 |
|---------------|----------|----------|
| `visual_profiles` | `VisualProfileRegistry` | `data/combat/visual_profiles` |
| `visual_cues` | `VisualCueRegistry` | `data/combat/visual_cues` |
| `visual_fx` | `VisualFxRegistry` | `data/combat/visual_fx` |
| `audio_cues` | `AudioCueRegistry` | `data/combat/audio_cues` |

这意味着素材包可以同时走两条兼容路径：`AssetRegistry` 按 `asset_index.json` 解析逻辑表现 ID，现有 registries 继续按 `register` 扫描 `.tres` 资源作为回退。

示例当前兼容 manifest：

```json
{
  "pack_id": "classic_original_assets",
  "enabled_by_default": false,
  "trust_level": "data_only",
  "register": ["visual_profiles", "visual_cues", "visual_fx", "audio_cues"],
  "capabilities": ["visual_profiles", "visual_cues", "visual_fx", "audio_cues"]
}
```

## 本地私有包根目录

正式设计新增本地私有扩展根：

```text
res://local_extensions
```

用途：

- 存放本机私有素材包。
- 存放原版素材导出产物。
- 存放依赖私有素材的 collection pack。

约束：

- `local_extensions/` 必须进入 `.gitignore`。
- 默认不作为发布物。
- 加载器应支持扫描该目录，但可通过 manifest 或 CLI 控制启用。
- CI 默认不依赖该目录存在。

当前代码已扫描 `res://local_extensions`；目录不存在时静默跳过，存在时按普通扩展包 manifest 规则加载。

已落地的本地私有原版素材包采用 ignored 嵌套 Git 仓库：

```text
local_extensions/
  classic_original_assets/
    .git/
    extension.json
    sources/
    generated/
    actors/
    data/combat/visual_profiles/
```

主仓只忽略 `local_extensions/` 并扫描该根；私有包内部自己管理 Git 历史，不作为 submodule，也不在主仓记录远端地址。这样可以在本机保存原版源素材和生成物，同时避免公开仓库泄漏不可发布内容。

`classic_original_assets` 的 manifest 保持 `enabled_by_default = false`，发布语义上仍然不是公开默认包。当前项目尚未对外发布，为了让主页面和本地运行时直接显示已迁移原版视觉，`project.godot` 暂时打开 `openpvz/debug/enable_classic_original_assets = true`；准备发布或跑无私有素材环境时应先关掉该 debug 开关。

本地私有验证仍通过 `local_private` layer 显式运行。`tools/validation_scenarios.json` 支持 `local_private` layer 与 `extra_user_args`，默认批量回归会跳过 `local_private` 场景；显式请求该 layer 时仍会把 `--include-classic-original-assets` 等私有启用参数传给 Godot：

```powershell
pwsh tools/run_all_validations.ps1 -Layers "local_private" -MaxParallel 1
pwsh tools/run_validation.ps1 -Scenario "res://scenes/validation/visual_private_classic_asset_pack_smoke.tres" -ExtraUserArgs "--include-classic-original-assets"
```

当前 `visual_private_classic_asset_pack_smoke` 会验证 `classic_original_assets` 被显式启用或被项目 debug 开关启用后，`AssetRegistry` 能从 `asset_index.json` 解析 Peashooter、Sunflower、ThreePeater、Chomper 和 Squash 的本地私有 visual profile，同时 `VisualProfileRegistry` 保持回退可用。

当前 `visual_private_classic_archetype_binding_smoke` 会进一步验证这些 profile 已正式写回对应 `CombatArchetype.visual_profile_id`，并且 `EntityFactory` 在私有包启用时会给 Peashooter、Sunflower、ThreePeater、Chomper 和 Squash 实体挂载 `VisualActorComponent`。关闭 debug 开关且不带 `--include-classic-original-assets` 时，主仓仍不加载私有包，实体会跳过私有视觉挂载。

当前 `tools/check_original_plant_showcase_visual_binding.gd` 会验证主页面原版植物展示的三组 showcase：已绑定的 5 株替换为真实 actor，未绑定样本仍保留占位，并额外检查 actor 有可见纹理、组件跟随实体、可见底部没有漂在实体原点上方。私有素材包内的 profile 通过 `ground_offset` 校准原版 actor 的像素落点，展示场景不写专用偏移。

## 原版迁移包拆分

推荐结构：

```text
extensions/
  openpvz_classic_semantics/
  openpvz_classic_content/
  openpvz_placeholder_assets/

local_extensions/
  classic_original_assets/
  classic_original_collection/
```

### openpvz_classic_semantics

公开发布。

内容：

- `.reanim` 导入 manifest 模板
- Action Recipe / Part Slot 语义模板
- 迁移文档
- 不含原版素材和派生产物

### openpvz_classic_content

可公开发布。

内容：

- 原版风格玩法内容定义
- 只引用逻辑表现 ID
- 不引用 `vendor/out_files` 或本地素材路径

### openpvz_placeholder_assets

公开发布。

内容：

- 合法占位 actor
- 占位卡片图标
- 占位 vfx/audio
- 作为无私有素材时的回退

### classic_original_assets

本地私有，不发布。

内容：

- 原版 `.reanim/.png/.ogg` 的导出产物
- `actor.tscn`
- `visual_profile.tres`
- `visual_fx.tres`
- `audio_cue.tres`
- `import_report.json`

### classic_original_collection

本地私有，不发布。

内容：

- 组合 `openpvz_classic_content` 与 `classic_original_assets`
- 提供本机“原版风格体验”入口

## Action Recipe 在素材包中的位置

当前 Action Recipe 仍处于迁移验证阶段，保存在 `tools/reanim_importer/reanim_visual_manifest.json` 中。

正式路线：

1. 过渡期：Action Recipe 继续作为本地导入 manifest 字段。
2. 稳定后：抽出 `VisualActionRecipeDef`。
3. 再稳定后：新增 `visual_action_recipes` register kind 和 registry。
4. 最终 asset_pack 通过 `visual_profiles + visual_action_recipes + visual_fx + audio_cues` 组合完整表现。

第一版不应设计完整时间轴语言，只收敛少量必要 op：

- `play_state`
- `play_action`
- `part_action`
- `root_motion`
- `emit_local_cue`
- `spawn_world_fx`
- `return_home`
- `hide_actor`

## 发布守卫

正式发布前必须满足：

- 公开包不得包含原版 `.reanim/.png/.ogg`。
- 公开包不得包含由原版素材生成的 actor、贴图、音频等派生产物。
- 公开包不得引用 `vendor/out_files`。
- 公开包不得引用 `local_extensions`。
- CI 不应依赖 `local_extensions` 存在。
- 本地私有包必须通过 `.gitignore` 和 manifest `publish_policy = local_private` 双重标记。

---

## 最小落地顺序

1. 文档正式化：固定包边界、本地私有根、发布守卫。
2. 增加 `local_extensions/` ignore 与扫描根。
3. 建立本地 ignored 嵌套 Git 仓库 `local_extensions/classic_original_assets`，把当前 `vendor/out_files` 源素材与 `_openpvz_import` 生成物迁入。
4. 公开 `openpvz_classic_semantics`，只放导入 manifest 模板和语义配置。
5. 为本地私有包提供显式 `local_private` 验证层，默认公开回归跳过该层。
6. showcase/validation 逐步改为通过 `AssetRegistry` / `VisualProfileRegistry` 的 profile id 加载；一旦场景声明 `profile_id`，就不得回退到旧本地路径，未启用素材包时应明确提示失败。
7. 先以 `AssetIndexCatalog + asset_index.json + AssetRegistry.resolve_visual_profile()` 固化运行时资产索引；后续再扩展到主题覆盖、FX、音频和完整回退链。
