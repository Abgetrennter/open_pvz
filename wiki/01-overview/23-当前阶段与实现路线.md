# 当前阶段与实现路线

- 状态：当前路线

> 本文是唯一的状态快照页，只回答“当前仓库已经成立什么、主线现在正在推进什么”。更偏前瞻的优先级排序请看 [开发路线图](../04-roadmap-reference/26-开发路线图.md)。

---

## 当前一句话

> Mechanic-first 重构三阶段已经完成；实体旧作者模型已归档，战斗模式组织层 v1 已进入可验证基线，`Placement` family 已完成正式收口，主线已进入“archetype-only 内容与回归同步”的阶段。

---

## 当前已经成立的事实

### 1. 正式作者模型已经切换

- 当前唯一正式作者入口是 `CombatArchetype + CombatMechanic[]`。
- 正式运行时唯一入口是 `CombatArchetype + CombatMechanic[] -> RuntimeSpec -> EntityFactory`。
- 实体旧作者模型已归档为历史记录，不再作为可运行兜底、后端资源层或扩展入口。
- `wiki/decisions/ADR-002` 的结论已经成为正文默认口径。

### 2. 重构三阶段已经完成

- 第一阶段：完成 archetype 入口、编译链骨架和首批 archetype 闭环。
- 第二阶段：完成 `Controller / State / Lifecycle` 的最小正式路径。
- 第三阶段：完成多 payload、per-type dispatch、确定性随机、独立实例化与迁移对照验证。

### 3. 当前资源与验证基线已形成

- `data/combat/archetypes/` 当前共 `96` 个 archetype。
- `tools/validation_scenarios.json` 当前共 `107` 个验证场景。
- 扩展包 smoke / chaos / guardrail 已进入稳定回归。
- battle 模式组织层 v1 已落地：`BattleModeHost / BattleModeDef / BattleRuleModule / BattleInputProfile / BattleObjectiveDef` 已接入运行时，并拥有 `manual aim` 试点、input guard、score objective、module override、conveyor 与 no-mode/protocol guardrail 验证。

### 4. Mechanic family 已冻结并进入完整正式编译覆盖理解

- 一级 family 已固定为 `10` 个：`Trigger / Targeting / Emission / Trajectory / HitPolicy / Payload / State / Lifecycle / Placement / Controller`。
- 正文当前按 `10/10` family 已建立正式编译覆盖理解。
- `Placement` 当前已具备专属 compiler callable、`placement_spec` 编译产物、`BattleCardState / BattleBoardState` 消费主链、`archetype_field` fallback 验证，以及多 Placement mechanic guardrail 验证。

### 5. 实体旧模型已归档

- 当前 archetype 资源不再携带 `legacy` 或 backend template 身份字段。
- `RuntimeSpec` 的触发器编译产物是 `RuntimeTriggerSpec[]`，不是实体旧绑定资源。
- 现阶段的工作重点是保持 archetype-only 内容、验证和扩展模板一致。

---

## 当前主线正在推进什么

### 1. 保持旧模型归档边界

旧模型只允许出现在历史 ADR、归档计划或迁移背景叙述中；正式资源、验证场景、扩展模板和运行时代码不得再依赖旧字段或旧资源脚本。

### 2. 保持内容、验证、展示同步

新增 archetype、扩展包能力或正式内容时，默认要同步检查：

- `tools/validation_scenarios.json`
- `tools/formal_content_validation_map.json`
- 对应 Wiki 正文
- battle mode 的新变化应优先落在 `BattleModeHost / BattleRuleModule / BattleInputProfile / BattleObjectiveDef`，而不是重新回流到 `BattleManager` 特判。

---

## 当前不应重新抬成主线的方向

- 重新回到“第七阶段输入准备”口径推进正文。
- 把 `EntityTemplate / TriggerBinding` 重新写成正式作者入口。
- 把双轨期并行继续写成长期正式策略。
- 在外围产品壳、完整编辑器或大规模 UI 包装上提前分散主线注意力。

---

## 当前状态下的判断标准

如果一个判断和下面任一事实冲突，应以本页口径为准：

- 新内容默认走 `Archetype + Mechanic[]`。
- 旧模型默认是归档历史，不是兼容层职责。
- 当前主线是保持 family 覆盖、运行时规格和内容验证的一致性。
- 任何正文中出现旧实体模型术语，都必须明确它是历史记录。
