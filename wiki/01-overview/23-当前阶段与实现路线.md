# 当前阶段与实现路线

> 本文只回答"当前主线应该做什么"。关于"当前每一层处于什么状态"的完整盘点，请读 [系统版图与规划分层](34-Open%20PVZ%20系统版图与规划分层.md)，本文不再重复该叙事。

---

## 一、当前阶段一句话

> 第一到第六阶段已完成主干沉淀。项目已从"第七阶段输入准备"正式切换到 **Mechanic-first 重构阶段**。

含义：

- 旧路线（围绕 `EntityTemplate / TriggerBinding` 继续加厚内容）已通过 ADR-001 正式终止。
- 当前主线是完成 `Archetype + Mechanic[]` 新作者模型的落地，建立完整编译链，并通过迁移对照验证。
- 旧路线归档分支：`codex/archive-triggerbinding-first`。

决策详情见 [wiki/decisions/](../decisions/README.md)。

---

## 二、已完成的重构里程碑

### 第一阶段：编译链最小闭环（已完成）

- `CombatArchetype` / `PlantArchetype` / `ZombieArchetype` / `ProjectileArchetype` 资源类型落地
- `CombatMechanic` 10 个一级 family 冻结（`Trigger / Targeting / Emission / Trajectory / HitPolicy / Payload / State / Lifecycle / Placement / Controller`）
- `MechanicCompiler / NormalizedMechanicSet / RuntimeSpec` 编译链骨架
- `MechanicFamilyRegistry / MechanicTypeRegistry / MechanicCompilerRegistry` 注册中心
- `EntityFactory` 双路径（archetype 路径优先 + 旧 template 路径兜底）
- 5 个 archetype 骨架验证通过

### 第二阶段：Controller / State / Lifecycle 最小路径（已完成）

- `ControllerComponent` + `ControllerRegistry`（`core.bite` / `core.sweep`）
- `StateComponent` 时间触发 + 事件触发 transition
- `Lifecycle.on_spawned` / `on_place` 编译路径

### 第三阶段：编译链补完与扩展（已完成）

- Multi-payload 编译（一个 trigger 可驱动多个 payload）
- Per-type compiler dispatch（`MechanicCompilerRegistry` 可分发编译 callable）
- State 扩展：`core.growth` / `core.rage`
- Lifecycle 扩展：`core.on_armed` / `core.on_state_enter` / `core.on_expire` / `core.on_removed`
- Controller 策略增强：`DetectionRegistry` 集成 + blackboard
- 确定性随机协议：`GameState.battle_seed` + seed 派生 + `ShuffleBag` 工具类
- Mechanic-local runtime state
- Archetype 独立实例化（无 backend_entity_template 路径）
- 迁移对照验证（sunflower / peashooter / zombie parity）

---

## 三、当前主线应该做什么

### 1. 扩展已编译 family 的 type 覆盖面

当前已编译的 family 仅为 Trigger / Lifecycle / Payload / Controller / State。剩余 5 个 family（Targeting / Emission / Trajectory / HitPolicy / Placement）需要建立编译路径，完成攻击链拆分。

### 2. 将更多旧内容迁移到 archetype

当前 16 个 archetype 中大部分仍依赖 `backend_entity_template`。继续将正式内容迁移为独立 archetype。

### 3. 保持回归验证同步

新增机制或 archetype 时同步补验证场景、更新 `tools/validation_scenarios.json`、更新验证矩阵。

---

## 四、当前不应成为主线的内容

- 未编译 family 的全量实现（应分批，按攻击链拆分优先级推进）
- 扩展包向 Mechanic 层迁移（主仓编译链稳定后再做）
- 大型编辑器工具链 / 外围产品壳
