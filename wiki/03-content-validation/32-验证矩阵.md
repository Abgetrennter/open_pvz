## 正式内容映射规则

当前正式内容与验证的关系统一按下面原则理解：

### 1. 正式内容定义归属

正式内容归属以：

- `data/combat/`
- `extensions/`
- `tools/formal_content_validation_map.json`

为准，而不是以 showcase 场景为准。

### 2. validation 负责证明可回归

validation 的职责是：

- 证明正式内容可运行
- 证明关键交互已进入回归
- 证明边界守卫长期有效

### 3. showcase 只负责展示和导航

showcase 不负责：

- 定义正式内容目录
- 判断能力是否已进入回归
- 替代验证矩阵

## 当前维护规则

### 1. 事实冲突时，以 JSON 为准

如果 Wiki 与 `validation_scenarios.json` 或 `formal_content_validation_map.json` 冲突，以 JSON 文件为准。

### 2. 新增正式内容默认补正向验证

只要进入主仓正式内容，默认先判断：

- 是否需要新增 `core` 验证
- 是否需要更新正式内容映射分组

### 3. 新增扩展能力默认补 guardrail

扩展能力不只补正向样例，还应考虑：

- manifest 边界
- registry 边界
- schema / slot / strategy 边界

### 4. 归档边界验证不跑旧路径

旧模型归档后，验证不再以“旧路径仍可运行”为成功条件；负向守卫应证明旧字段无法回流正式内容。
