## 当前验证体系概览

当前验证体系的事实来源是：

- [`tools/run_validation.ps1`](../../tools/run_validation.ps1)
- [`tools/run_all_validations.ps1`](../../tools/run_all_validations.ps1)
- [`tools/validation_scenarios.json`](../../tools/validation_scenarios.json)

- 当前共 `87` 个验证场景
- 支持 `smoke / core / extension / guardrail / migration` 五类分层
- 默认启动场景仍是 [`scenes/main/main.tscn`](../../scenes/main/main.tscn)，但验证应优先通过脚本入口运行

## 当前固定入口

### 1. 交互展示入口

用于人工试玩和场景浏览：

- [`scenes/main/main.tscn`](../../scenes/main/main.tscn)
- [`scenes/demo/demo_level.tscn`](../../scenes/demo/demo_level.tscn)
- [`scenes/showcase/README.md`](../../scenes/showcase/README.md)

这类入口适合：

- 观察交互体验
- 检查 UI 路径
- 快速肉眼确认 Showcase 是否还能进入

不适合作为回归真入口。

### 2. 单场景验证入口

用于跑一个指定 `.tres` 验证资源：

- [`tools/run_validation.ps1`](../../tools/run_validation.ps1)

脚本默认场景当前是：

- `res://scenes/validation/parabola_long_range_validation.tres`

### 3. 批量回归入口

用于跑整套或分层验证集：

- [`tools/run_all_validations.ps1`](../../tools/run_all_validations.ps1)
- [`tools/validation_scenarios.json`](../../tools/validation_scenarios.json)

### 4. 正式内容映射入口

用于看正式内容分组和对应验证：

- [`tools/formal_content_validation_map.json`](../../tools/formal_content_validation_map.json)

## 当前推荐命令

### 1. 单场景验证

```powershell
& ".\tools\run_validation.ps1"
```

### 2. 指定验证资源

```powershell
& ".\tools\run_validation.ps1" -Scenario "res://scenes/validation/minimal_battle_validation.tres"
```

### 3. 批量跑全部验证

```powershell
& ".\tools\run_all_validations.ps1"
```

### 4. 按层跑 smoke

```powershell
& ".\tools\run_all_validations.ps1" -Layers smoke
```

### 5. 按层跑 extension 和 guardrail

```powershell
& ".\tools\run_all_validations.ps1" -Layers extension,guardrail
```

### 6. 打开运行时快照

```powershell
& ".\tools\run_validation.ps1" -Scenario "res://scenes/validation/parabola_long_range_validation.tres" -EnableRuntimeSnapshots
```

## 当前产物位置

单场景执行后，脚本会在 `artifacts/validation/` 下生成一轮运行目录，常见文件包括：

- `godot.log`
- `validation_report.json`
- `debug_logs.json`

批量执行后，除了批次目录内的 `summary.json / summary.txt`，还会维护：

- `latest_summary.json`
- `latest_summary.txt`
- `regression_history.jsonl`
- `regression_status.json`
- `regression_status.md`

---

## 跑验证时优先看什么

### 1. `validation_report.json`

先看最终状态和失败原因，不要第一时间只看场景表现。

### 2. `godot.log`

看：

- `protocol.issue`
- 关键事件链
- 脚本报错
- 场景切换失败

### 3. `debug_logs.json`

当问题涉及位置、高度、目标选择或状态变化时，优先看结构化调试日志。

### 4. 回归汇总文件

批量执行后优先看：

- `summary.json`
- `regression_status.json`
- `regression_status.md`

---

## 当前维护规则

- 验证事实以脚本和清单为准，不以 Wiki 手写枚举为准。
- 新增正式内容时，优先考虑是否需要补一个 `core` 或 `smoke` 场景。
- 新增扩展能力时，优先考虑是否需要补一个 `extension` 场景和一个 `guardrail` 场景。
- `scenes/showcase/` 只负责展示，不应替代正式验证入口。
