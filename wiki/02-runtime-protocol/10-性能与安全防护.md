# 性能与安全防护

- 状态：当前事实


> 这篇文档只记录当前主干里已经落地或已明确采用的约束，不再把早期草案、Unity/C# 示例和未来设想写成当前事实。

---

## 文档定位

这篇文档主要回答：

- 当前主干已经有哪些硬边界
- 当前有哪些日志和历史上限
- 当前哪些保护措施已经进入运行时
- 哪些“性能/安全”想法目前还没有真正落地

这篇文档不负责：

- 给未来所有优化方向列愿望单
- 假装当前已经有完整脚本沙盒或全量协议校验

---

## 当前状态

### 已决定

- 规则执行必须带深度边界。
- 调试日志和事件历史都必须有限长。
- 连续行为必须能在失败时被定位，而不是静默漂移。
- 运行时保护优先服务“可解释、可验证”，而不是先追求复杂优化。

### 当前已形成的治理配套

- 验证矩阵已经成为正式主干保护设施
- 术语表和文档治理入口已经形成固定接口
- 批量验证入口已经进入默认工作流

### 当前仍未进入主干

- 协议兼容 / 废弃清单仍未形成单独正式页
- 更系统的性能剖析工具仍未进入仓库主干

## 当前已经实现的硬边界

### 1. 效果链深度上限

当前实现：

- [`scripts/core/runtime/effect_executor.gd`](../../scripts/core/runtime/effect_executor.gd)

当前规则：

- `EffectExecutor.MAX_DEPTH = 5`
- 当 `context.runtime.depth` 或递归深度超过上限时，当前效果执行会被终止并返回失败结果

这是真正已经进入代码的边界，不是未来建议。

### 2. 事件历史上限

当前实现：

- [`autoload/EventBus.gd`](../../autoload/EventBus.gd)

当前规则：

- `EventBus.MAX_HISTORY = 256`

这保证了事件历史可回看，但不会无限增长。

### 3. DebugService 日志上限

当前实现：

- [`autoload/DebugService.gd`](../../autoload/DebugService.gd)

当前规则：

- `MAX_EVENTS = 128`
- `MAX_EFFECTS = 128`
- `MAX_TRIGGERS = 128`
- `MAX_RUNTIME_SNAPSHOTS = 128`

这保证了调试数据不会无限堆积。

### 4. 投射体消费与终止保护

当前实现：

- [`scripts/entities/projectile_root.gd`](../../scripts/entities/projectile_root.gd)

当前规则包括：

- 命中后 `_consumed = true`
- 不会重复命中自己、来源节点或同队目标
- 终点解析统一走 `_resolve_terminal_state()`
- 默认 `lifetime = 5.0`
- 同时只保留一个 movement component

### 5. 实体生命边界

当前实现：

- [`scripts/components/health_component.gd`](../../scripts/components/health_component.gd)

当前规则：

- `current_health <= 0` 后不会继续受伤
- 死亡时统一推送 `entity.died`

### 6. 验证时间窗口

当前实现：

- [`scripts/battle/battle_manager.gd`](../../scripts/battle/battle_manager.gd)
- [`scripts/battle/battle_scenario.gd`](../../scripts/battle/battle_scenario.gd)

当前规则：

- 每个 `BattleScenario` 可定义 `validation_time_limit`
- 超时后验证状态会转为 `failed`
- 可通过命令行自动退出并返回退出码

## 当前已经实现的运行时保护

### 1. 结构化事件默认值补全

当前实现：

- [`scripts/core/runtime/event_data.gd`](../../scripts/core/runtime/event_data.gd)

当前保护包括：

- 自动补全 `event_name`
- 自动生成 `event_id`
- 自动继承或生成 `chain_id`
- 自动规范化 `depth`
- 自动写入 `timestamp`

### 2. 订阅生命周期保护

当前实现：

- [`scripts/components/trigger_component.gd`](../../scripts/components/trigger_component.gd)

当前保护包括：

- 重新绑定触发器时先退订旧订阅
- 节点退出时自动清理订阅

### 3. 高度命中保护

当前实现：

- [`scripts/entities/base_entity.gd`](../../scripts/entities/base_entity.gd)
- [`scripts/entities/projectile_root.gd`](../../scripts/entities/projectile_root.gd)

当前保护包括：

- 每个目标有命中高度范围
- 投射体命中判断要求高度区间重叠

这部分现在既是空间模型的一部分，也是防错设施的一部分。

### 4. 运行时快照保护

当前实现：

- [`autoload/DebugService.gd`](../../autoload/DebugService.gd)
- [`scripts/battle/battle_manager.gd`](../../scripts/battle/battle_manager.gd)

当前能力：

- 按帧间隔记录运行时实体快照
- 输出当前实体状态、位置、生命和 values

这不是单纯调试功能，它也是当前排查空间与时序问题的重要保护设施。

## 当前性能策略的真实状态

当前项目的性能策略，应当诚实地理解为：

### 已落地

- 有界历史
- 有界日志
- 有界效果深度
- 有界验证窗口
- 可按需启停的快照日志

### 还没落地成正式主干

- 对象池
- 全局事件合并
- 大规模惰性资源卸载策略
- 专门的运行时 profiler 面板

这些内容可以在未来补，但当前不要继续把它们写成已实现能力。

---

## 当前“安全”语义的真实边界

在这个项目里，“安全防护”当前主要指的是：

- 规则链不要无限扩张
- 事件链能够被追踪
- 失败能够被定位
- 空间和命中错误能够被验证出来

它当前不等于：

- 通用脚本沙盒
- 进程级隔离
- 网络/文件系统权限模型

如果后续要讨论更强的脚本安全模型，应另开文档，并明确标记为未来方向。
