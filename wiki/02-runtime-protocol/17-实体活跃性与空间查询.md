# 实体活跃性与空间查询

- 状态：当前事实

> 本文说明当前规则运行时如何表达“实体是否参与某类交互”，以及目标查询如何通过 `SpatialIndex` 收口。它取代旧的单一 `combat_active` 口径。

---

## 核心判断

Open PVZ 不再使用一个布尔值描述实体活跃性。

当前基线是：

- 实体自身能力由多维 liveness profile 表达。
- 状态与 status 通过 liveness override 临时改变维度。
- 查询层只处理空间、分组和高度过滤。
- 命中资格由调用方使用 `target_exposure_states` 明确表达，默认只命中 `ground`。
- 业务语义由调用方通过明确 API 或 `filter` 决定。

这样可以避免“睡眠、埋地、冻结、待机、死亡统计、碰撞、受伤、触发器是否运行”全部挤进同一个开关。

---

## Liveness 维度

当前 `BaseEntity` 暴露的关键语义包括：

| 语义 | 用途 |
|------|------|
| `is_runtime_alive()` | 节点是否仍是运行时有效实体 |
| `is_targetable()` | 能否被索敌或选作攻击目标 |
| `is_damageable()` | 能否承受伤害 |
| `is_collidable()` | 能否参与碰撞或接触结算 |
| `is_counted_for_objectives()` | 是否参与胜败/目标统计 |
| `is_liveness_enabled(&"movement")` | movement 是否可推进 |
| `is_liveness_enabled(&"controllers")` | controller 是否可执行 |
| `is_liveness_enabled(&"triggers")` | trigger 是否可响应 |

这些语义可以组合，但不应互相替代。

示例：

- Potato Mine arming：不可被索敌、不可受伤、不可碰撞，但仍是运行时实体。
- sleeping mushroom：trigger/controller 默认关闭，wake 后恢复。
- stunned status：关闭 movement/controller，但不关闭 damageable。
- 割草机 field_object idle：可以占场并由 `Controller.core.sweep` 等待触发，但不等同于可攻击单位。

## 状态与 Status

`StateComponent` 当前使用统一转换入口：

- 时间转换和事件转换都进入 `_execute_transition()`。
- 转换时同步处理 exit / enter / rejected 事件。
- 状态进入和退出时同步 push / pop 对应 liveness override。

可观测事件：

| 事件 | 说明 |
|------|------|
| `entity.state_entered` | 进入状态 |
| `entity.state_exited` | 离开状态 |
| `entity.state_transition_rejected` | 转换条件不满足 |
| `entity.liveness_changed` | 合成后的 liveness 维度发生变化 |

Status 不再使用 `blocks_attack`。需要暂停行为时，应通过 `liveness_overrides` 表达，例如关闭 `movement`、`controllers` 或 `triggers`。

## 暴露态与重量

`BaseEntity` 当前同步两组与命中资格相关的运行时字段：

| 字段 | 默认值 | 说明 |
|------|--------|------|
| `exposure_state` | `ground` | 当前可被哪些攻击规则命中 |
| `weight_class` | `normal` | 推力、驱散、击退等效果的重量过滤依据 |

已冻结的暴露态：

- `ground`
- `airborne`
- `flying`
- `submerged`
- `underground`

已冻结的重量级别：

- `light`
- `normal`
- `heavy`
- `massive`
- `fixed`

`CombatArchetype.initial_exposure_state` 和 `weight_class` 会经 `RuntimeSpec` 进入 `EntityFactory`，并写入 `EntityState` 快照。Movement 或 State side-effect 改变高度/接地状态时，也必须同步 `exposure_state`，不能只改屏幕位置。

默认命中规则：

- `Effect.damage`、`Effect.explode`、projectile candidate 与 zombie direct attack 默认 `target_exposure_states=["ground"]`。
- 能打 `flying`、`airborne`、`submerged` 或 `underground` 的内容必须显式 opt-in。
- `HealthLayer` 只负责伤害承接顺序，不决定“能不能命中”；`attachment` layer 破坏后可以通过 State side-effect 改变暴露态。

## SpatialIndex 职责

`SpatialIndex` 是 battle 内部的查询基础设施，对外入口是：

```gdscript
BattleManager.spatial_query(params)
```

它负责：

- 按 team、lane、tag、kind 建立基础分组。
- 对 x 范围、radius、center 做空间过滤。
- 对 `height_range` 做 HeightBand overlap。
- 提供稳定排序和 `max_results` 限制。

它不负责：

- 判断目标是否可被攻击。
- 判断目标是否可受伤。
- 判断实体是否参与胜败统计。
- 缓存 liveness 业务语义。
- 替代 trigger、effect、controller 的具体规则。

---

## 查询参数口径

常见参数：

| 参数 | 说明 |
|------|------|
| `team_include` / `team_exclude` | 队伍过滤 |
| `lane_ids` | 限定车道 |
| `x_min` / `x_max` | 横向范围 |
| `center` / `radius` | 圆形范围 |
| `tags_any` | 至少命中一个 tag |
| `kind_include` | 限定实体 kind |
| `height_range` | 查询高度范围，与目标 hit height overlap |
| `sort_by_distance` | 按距离排序 |
| `max_results` | 限制返回数量 |
| `filter` | 调用方注入业务判断 |

业务过滤示例：

```gdscript
var targets: Array = battle.spatial_query({
	"team_exclude": source_team,
	"lane_ids": PackedInt32Array([lane_id]),
	"x_min": source_x,
	"filter": func(candidate):
		return candidate.has_method("is_targetable") and candidate.call("is_targetable")
})
```

## 热路径使用约定

新增目标查询时默认遵守：

- Detection：使用 `is_targetable()`。
- Projectile overlap / segment：使用 `is_collidable()`，再做精确几何确认。
- Projectile terminal / area：按 hit strategy 使用 `height_range` 和半径。
- HitPolicy / Effect：默认额外按 `target_exposure_states` 过滤。
- Zombie 攻击目标：使用 `is_targetable()`，实际伤害仍由 `HealthComponent` / `is_damageable()`确认。
- Effect damage radius：使用 `is_damageable()`。
- Controller bite / sweep：根据 controller 语义选择 `is_targetable()` 或 `is_collidable()`。
- Objective / WaveRunner：使用 `is_counted_for_objectives()`。

`get_runtime_combat_entities()` 只保留为 `SpatialIndex.rebuild()` 数据源、验证基线和调试工具，不应作为新热路径查询入口。

---

## 验证入口

当前专项验证：

- `liveness_system_validation`
- `spatial_index_consistency_validation`
- `tick_budget_monitor_validation`
- `hit_policy_exposure_ground_default_validation`
- `hit_policy_exposure_flying_validation`
- `hit_policy_exposure_hidden_validation`
- `force_weight_filter_validation`

维护要求：

- 新增 liveness 维度或状态 override 时，补 liveness 专项断言。
- 新增空间查询参数时，补 SpatialIndex 与全量基线一致性断言。
- 新增命中高度语义时，补 `height_range` 组合过滤验证。
- 新增暴露态或重量过滤语义时，补 `target_exposure_states` 与 weight 专项验证。

---

## 不做事项

- 不重新引入 `is_combat_active()`。
- 不在 `SpatialIndex` 中缓存业务活跃性。
- 不新增通用 collision matrix，除非真实内容需要同队内复杂碰撞规则。
- 不把视觉表现状态写入 liveness。
