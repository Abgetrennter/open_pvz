# 连续行为模型

- 状态：当前事实


> 这篇文档说明当前主干如何处理持续变化对象，尤其是投射体。它已经不再使用早期“纯 `velocity / position` 贡献项 + JSON 轨迹注册”的叙事，而是对齐当前已经落地的 `ground_position + height + projected_position` 模型。

---

## 文档定位

这篇文档主要回答：

- 当前项目如何表达连续行为
- 为什么投射体现在采用“逻辑 3D + 2D 投影”
- 当前移动组件如何返回统一的运动结果
- 当前命中判定如何基于地面平面和高度规则运行

这篇文档不负责：

- 单独定义触发器或效果协议
- 单独定义验证清单

---

## 当前状态

### 已决定

- 当前投射体运行模型采用：
  - `ground_position`
  - `height`
  - `projected_position`
- 命中判定以“地面平面 + 高度范围”为主，不再依赖屏幕投影重叠。
- 投射体移动组件统一返回 `ProjectileMoveResult`。
- `HeightBand` 和 `ProjectileFlightProfile` 已经进入正式 `Resource` 体系。

### 当前建议

- 后续继续把空间模型和高度规则整理成单独协议文档。
- 继续让更多命中策略和高度策略显式资源化。

### 未来方向

- 更完整的对空 / 对地规则
- 更细的高度策略和飞行单位体系
- 更强的路径调试与可视化

---

## 当前连续行为的两条主线

当前项目里，连续行为有两条不同层级的实现。

### 1. 普通实体移动

普通实体当前仍可使用通用 `MovementComponent`：

- [`scripts/components/movement_component.gd`](../../scripts/components/movement_component.gd)

它适合：

- 僵尸前进
- 地面单位速度叠加

当前特点：

- 在 2D 地面平面上移动
- 支持 `velocity + contributions`
- 每帧把 `velocity / speed` 写回实体状态

### 2. 投射体移动

投射体当前使用专用 movement strategy：

- [`scripts/projectile/movement/projectile_movement_base.gd`](../../scripts/projectile/movement/projectile_movement_base.gd)
- [`scripts/projectile/movement/projectile_movement_linear.gd`](../../scripts/projectile/movement/projectile_movement_linear.gd)
- [`scripts/projectile/movement/projectile_movement_track.gd`](../../scripts/projectile/movement/projectile_movement_track.gd)
- [`scripts/projectile/movement/projectile_movement_parabola.gd`](../../scripts/projectile/movement/projectile_movement_parabola.gd)

它适合：

- 线性弹道
- 追踪弹道
- 抛物线弹道

这两条主线不要再混写成同一个“泛轨迹系统事实”。

## 当前空间模型

这是这篇文档最重要的部分。

当前投射体主干已经采用：

| 字段 | 作用 |
|------|------|
| `ground_position` | 逻辑地面位置，参与移动、选敌、命中 |
| `height` | 离地高度，参与投射与高度判定 |
| `projected_position` | 最终渲染到 2D 屏幕的位置 |

当前投影关系是：

```text
projected_position = ground_position + Vector2(0, -height * projection_scale)
```

当前落点：

- [`scripts/entities/projectile_root.gd`](../../scripts/entities/projectile_root.gd)
- [`scripts/entities/base_entity.gd`](../../scripts/entities/base_entity.gd)

---

## 当前命中模型

当前命中不再默认依赖屏幕上 `global_position` 的重叠。

当前命中判断由两部分组成：

### 1. 平面命中

当前支持：

- `swept_segment`
- `overlap`
- `terminal_hitbox`
- `terminal_radius`

其中最关键的是：

- 高速直线和追踪优先用线段扫掠
- 抛物线优先用终点命中

### 2. 高度命中

每个候选目标都有命中高度范围：

- `get_hit_height_range()`

默认来自：

- [`scripts/entities/base_entity.gd`](../../scripts/entities/base_entity.gd)
- [`scripts/core/defs/height_band.gd`](../../scripts/core/defs/height_band.gd)

当前判断规则是：

- 投射体飞行区间高度必须与目标 `min_height .. max_height` 重叠

这也是当前避免“抛物线飞过别的 lane 却误伤”的关键基础。

## ProjectileMoveResult

当前所有投射体移动组件统一返回：

- [`scripts/projectile/projectile_move_result.gd`](../../scripts/projectile/projectile_move_result.gd)

字段：

| 字段 | 作用 |
|------|------|
| `previous_position` | 上一帧地面位置 |
| `current_position` | 当前帧地面位置 |
| `previous_height` | 上一帧高度 |
| `current_height` | 当前帧高度 |
| `still_active` | 是否继续飞行 |
| `terminal_reason` | 结束原因 |

这意味着当前主干不再只是“组件返回 bool”，而是已经把：

- 运动段
- 高度段
- 终止原因

正式纳入了协议。

---

## 当前投射体移动类型

### 1. Linear

当前实现：

- [`scripts/projectile/movement/projectile_movement_linear.gd`](../../scripts/projectile/movement/projectile_movement_linear.gd)

特点：

- 按方向和速度推进 `ground_position`
- `height = 0`
- 默认命中策略以 `swept_segment` 为主

### 2. Track

当前实现：

- [`scripts/projectile/movement/projectile_movement_track.gd`](../../scripts/projectile/movement/projectile_movement_track.gd)

特点：

- 每帧根据目标地面位置修正方向
- `height = 0`
- 默认命中策略以 `swept_segment` 为主

### 3. Parabola

当前实现：

- [`scripts/projectile/movement/projectile_movement_parabola.gd`](../../scripts/projectile/movement/projectile_movement_parabola.gd)

特点：

- 在地面平面上做 `start -> target` 插值
- 在高度维上按抛物线计算 `height`
- 支持有限的动态终点修正
- 默认以终点命中为主，而不是飞行途中 overlap 命中

## ProjectileRoot 的职责

当前投射体根节点实现落在：

- [`scripts/entities/projectile_root.gd`](../../scripts/entities/projectile_root.gd)

它当前负责：

- 发射参数接收
- movement component 的唯一性维护
- `ground_position / height` 状态维护
- 飞行中命中判定
- 终点命中判定
- 生命周期结束处理
- `projectile.spawned / projectile.hit / projectile.expired` 事件回写

当前不应再把它理解成“纯视觉子弹节点”。

## 当前资源协议

### HeightBand

当前实现：

- [`scripts/core/defs/height_band.gd`](../../scripts/core/defs/height_band.gd)
- [`data/combat/height_bands/ground_unit.tres`](../../data/combat/height_bands/ground_unit.tres)

当前字段：

- `band_id`
- `min_height`
- `max_height`

它决定单位能在什么高度范围内被命中。

### ProjectileFlightProfile

当前实现：

- [`scripts/projectile/projectile_flight_profile.gd`](../../scripts/projectile/projectile_flight_profile.gd)
- [`data/combat/projectile_profiles/`](../../data/combat/projectile_profiles)

当前字段包括：

- `move_mode`
- `height_strategy`
- `peak_height`
- `projection_scale`
- `max_hit_height`
- `hit_strategy`
- `terminal_hit_strategy`
- `impact_radius`
- `collision_padding`
- `travel_duration`
- `lead_time_scale`
- `dynamic_target_adjustment`
- `dynamic_target_axis`

这意味着飞行策略和高度命中策略已经不再是纯代码特判，而是正式进入资源层。

## 当前非目标

下面这些内容不应再被写成当前连续行为模型已经成立的事实：

- 统一的 JSON 轨迹注册协议
- 完整的“贡献项叠加式”投射体轨迹组合框架
- 完整飞行单位和对空系统
- 完整 3D 物理场景

当前真正已经成立的是：

> 逻辑地面位置 + 逻辑高度 + 2D 投影 + 平面命中 + 高度命中
