# 第五阶段-v1现状盘点

> 这份文档用于完成 `第五阶段-v1规范化-4到8周执行路线图` 中的 `Epic A：现状盘点与文档收口`。它记录当前仓库真实状态、文档状态漂移、扩展包职责、回归入口数量，以及进入 `v1` 规范化前应采用的稳定性口径。

---

## 文档定位

这篇文档主要回答：

- 当前第五阶段真实推进到了哪里
- 哪些文档表述已经滞后于实现
- 当前已落地的扩展包各自负责什么
- 当前回归入口和场景数量是多少
- 哪些能力应优先保持稳定，哪些仍允许试验

这篇文档不负责：

- 代替 `v1` 协议冻结清单
- 承诺最终版扩展生态
- 展开第六阶段外循环实现
- 重新定义第五阶段全部任务

---

## 一句话判断

当前项目已经不只是“刚进入第五阶段”。

更准确的判断是：

> 第一到第四阶段已经形成稳定主干，第五阶段核心起步也已完成收口；当前主线应转向 `v1` 规范化，把已跑通的错误技、扩展入口、扩展 effect 和回归体系收敛成可持续演进的稳定边界。

## 能力稳定性分组

### A. 稳定优先

这些能力已经支撑大量验证和样例，后续应优先保持兼容：

- 结构化事件链
- `EffectExecutor` 执行协议
- `RuleContext` 基础上下文字段
- `EntityState` 基础运行时状态
- `EntityTemplate / ProjectileTemplate / TriggerBinding` 主链
- `BattleEconomyState / BattleBoardState / BattleCardState`
- `BattleFlowState / BattleStatusState / BattleFieldObjectState`
- 投射体线性、追踪、抛物线运行时
- 高度命中和终点命中模型
- `tools/validation_scenarios.json` 作为验证清单入口
- `minimal_chaos_pack` 的最小扩展包 smoke test
- `phase5_guardrail_pack` 的扩展 effect 守卫基线

### B. 谨慎演进

这些能力已经可用，但还需要在 `v1` 冻结前继续收紧：

- effect 参数 schema
- trigger 条件参数 schema
- `BattleSpawnEntry.spawn_overrides` 支持字段
- 扩展包 `extension.json` manifest 字段
- 扩展 effect 的 `strategy_script` 注册边界
- 扩展 effect 家族的命名和分类
- Showcase 与验证场景的一一映射口径
- 调试 overlay、runtime snapshot 和协议 issue 日志格式

### C. 可试验迭代

这些内容仍允许快速试验，但不应反向定义核心协议：

- 新扩展 effect 原子
- 新错误技样例组合
- 新 Showcase 包装场景
- 内容作者脚手架
- 回归分层脚本增强
- 外循环接口预留文档

### D. 后置规划

这些系统当前不进入实现主线：

- 完整随机错误技生成器
- 完整社区工坊
- 完整扩展安装器
- 完整编辑器产品化
- 完整选关 UI
- 完整存档、图鉴、商店、花园系统

## 当前禁止自由漂移的字段和入口

在 `v1` 协议冻结前，下面内容不应继续无约束新增或改名：

- 核心事件名
- `RuleContext` 的 source、target、chain、depth 语义
- `EntityTemplate` 的模板身份与运行时 trigger 绑定字段
- `ProjectileTemplate` 的投射体模板身份、飞行 profile 和默认参数字段
- `TriggerBinding` 的 `behavior_key / trigger_id / event_name` 对应关系
- `BattleSpawnEntry.spawn_overrides` 的允许键集合
- `EffectDef` 的 `effect_id / param_defs / slots / strategy_script` 语义
- `extension.json` 的 `pack_id / enabled_by_default / register / activation_*` 语义
- `tools/validation_scenarios.json` 的 `id / scenario / description` 基础结构

如果确实要新增，应同时满足：

1. 有文档说明为什么需要新增
2. 有 `ProtocolValidator` 或注册期守卫
3. 有正向验证
4. 有负向验证或失败口径
5. 已更新验证矩阵
