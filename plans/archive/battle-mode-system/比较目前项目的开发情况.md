基于对两个项目的深入对比分析，以下内容更适合被理解为 Open PVZ 在**支撑层规范化**上的主要待补方向，而不是当前主干推进的 P0 阻塞项。

需要先明确当前口径：

- 仓库已经完成第五、第六阶段核心输出，并进入第七阶段输入准备
- 输入、UI 和 demo 交互闭环并非空白，而是已经有 `scripts/demo/` 下的可用原型
- 当前讨论视觉、输入、音频和 UI，更准确地说是在收口“支撑层规范化方案”，而不是在判断主干是否还能继续推进

在这个前提下，下面几项仍然值得补齐，但应理解为**支撑层能力优先级**，不是“项目能否继续开发”的先决条件：

- 一、视觉表现层基础设施（最大缺口）
参考项目有完整的 AnimComponentBase + BodyCharacter 体系，支持动画状态机、速度缩放、方向翻转、颜色染色（冰冻/黄油）、阴影渲染。Open PVZ 目前实体是纯逻辑节点，没有任何视觉表现基础设施。

具体缺失：

动画状态机抽象 — 无 AnimComponent 或等效机制，植物/僵尸无法驱动动画状态（idle/attack/die 等）
精灵/贴图管理 — 无 sprite flipping、tint、shader 参数驱动的统一接口
阴影系统 — 参考项目有 Shadow 组件，Open PVZ 无任何深度/阴影基础设施
视觉反馈管线 — 受击闪烁、冰冻变色、死亡消散等效果没有从 status → 视觉的映射通道
影响： 这是当前支撑层中最值得优先规范化的一项。规则主干和正式内容已经能跑，但仍主要依赖 `_draw()` 占位和原型呈现。

- 二、音频系统（完全空白）
参考项目有 SoundManager 单例，支持：

对象池化 AudioStreamPlayer
分类管理（BGM / SFX / 角色 / 僵尸 / Dave）
帧级冷却防抖
总线音量控制
Open PVZ 完全没有音频基础设施，甚至没有 audio bus 布局的设计。

影响： 音频仍然是明显空白的表现层能力位，但其优先级应服从正式内容主线，而不是反向替代主线。

- 三、输入交互层
参考项目有完整的鼠标+键盘输入系统：

格子点击放置植物
阳光点击收集
卡片槽选择 + 键盘快捷键（1-9）
铲子工具
Open PVZ 的 BattleCardState 有放置请求的状态逻辑，但缺少从用户输入到状态变更的交互通道。

影响： 当前已有 demo 级输入闭环，但仍缺一套可复用、可验证、可脱离 demo 目录的规范化输入层。

- 四、UI 框架层
参考项目有卡片 UI、阳光计数器、波次进度条、暂停菜单等完整 UI 体系。Open PVZ 当前已有 `scripts/demo/` 下的原型 UI，但还没有收口成正式框架层。

需要建立的基础设施（不是具体 UI 内容）：

UI 信号总线或 UI ↔ BattleManager 的标准通信接口
面板/控件的基类抽象（卡片槽、资源计数器、进度条）
HUD 生命周期管理（随战斗创建/销毁）
影响： UI 层的设计决定了状态系统如何对外暴露，属于当前阶段值得推进的支撑层规范化方向。

- 五、实体生命周期与对象池
参考项目有 ObjectPool 模式，用于 AudioStreamPlayer 和场景实例复用。Open PVZ 的 EntityFactory 每次 instantiate() 创建新实例，无回收机制。

在高频场景（大量抛射体、密集生成/销毁）下，缺少池化会导致：

GC 压力
创建开销
帧率抖动
建议： 至少为 ProjectileRoot 实现基本的对象池化，这是最容易验证且收益最高的优化点。

- 六、场景流程与持久化基础
参考项目有 SaveService（JSON 持久化、版本迁移、自动保存）和场景过渡系统。Open PVZ 完全缺失：

场景过渡机制（主菜单 → 关卡选择 → 战斗 → 结果）
存档数据模型和序列化接口
关卡进度追踪
优先级相对较低（当前是引擎验证阶段），但需要预留接口位置，否则后续接入会涉及大量子系统改造。

总结：支撑层优先级排序

优先级|基础设施|理由
S1|视觉表现层（动画/精灵/反馈）|当前最值得优先规范化的表现层能力位
S1|输入交互层|当前已有 demo 级闭环，但缺正式输入路由和观测路径
S2|音频系统|事件→音效的映射值得尽早设计，但不应抢正式内容主线
S2|UI 框架层|已有原型，下一步是抽象、生命周期和状态绑定收口
P2|对象池化|性能优化，当前规模不急但架构上需预留
P2|场景流程/持久化|引擎验证阶段非必须，但需预留接口
核心判断： Open PVZ 的规则引擎骨架和正式战斗内容主链已经相当成熟，但表现层与交互层仍主要停留在原型和占位态。下一步更适合把这些方向作为支撑层规范化方案逐步收口，让正式内容更可见、更可操作、更可维护。
