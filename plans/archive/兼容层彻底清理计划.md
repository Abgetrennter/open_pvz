# Legacy 兼容层彻底清理计划

## Summary

本计划按“允许破坏旧资源兼容”的口径执行：`EntityTemplate / TriggerBinding` 不再是可运行兜底、后端资源层或编译产物，只保留为归档历史。正式运行时唯一入口是 `CombatArchetype + CombatMechanic[] -> RuntimeSpec -> EntityFactory`。

保留 `ProjectileTemplate`，但仅作为抛射体内容资源；不得再把它和实体 legacy 作者模型并列描述。

## Key Changes

- 删除实体旧模型：
  - 删除 `EntityTemplate`、`TriggerBinding` 脚本定义、preload、校验函数、扩展模板、迁移脚本和旧展示入口。
  - `ProtocolValidator` 不再校验旧资源，只拒绝任何旧字段出现在正式内容中。
  - 旧 `.tres` 资源只允许留在 `plans/archive/legacy-resources/` 或新的历史归档目录，且不被 `SceneRegistry`、工具模板、验证清单扫描。

- 改运行时编译产物：
  - 新增 `RuntimeTriggerSpec`，字段固定为：`spec_id`、`trigger_id`、`event_name`、`condition_values`、`effect_root`。
  - `RuntimeSpec.compiled_trigger_bindings` 改为 `trigger_specs`。
  - `MechanicCompiler` 直接产出 `RuntimeTriggerSpec`，不再构造 `TriggerBinding`。
  - `EntityFactory` 改为 `build_runtime_triggers_from_specs()`，从 spec 创建 `TriggerInstance`。

- 移除实体 `template_id` 身份链：
  - `CombatArchetype.legacy_template_id` 删除，95 个 archetype 资源同步移除该字段。
  - `BaseEntity`、`EntityState`、`EventData`、`TriggerInstance` 删除实体 `template_id`、`source_template_id`、`target_template_id`、`legacy_template_id`。
  - 实体事件与验证统一使用 `source_archetype_id`、`target_archetype_id`、`archetype_id`。
  - 抛射体如需身份，只使用 `projectile_template_id`，不复用实体 `template_id`。

- 改 battle/mode 协议：
  - `protected_template_id` 改为 `protected_archetype_id`。
  - objective 参数中 `template_id / target_template_ids` 改为 `archetype_id / target_archetype_ids`。
  - `BattleModeInputRequest.legacy_template_id` 删除，只允许 `entity_id` 或 `entity_archetype_id`。
  - field object 事件删除 `object_template_id`，改为 `object_archetype_id`。

- 清理内容、验证和文档：
  - 所有验证场景从 template 字段迁移到 archetype 字段；涉及 projectile 的断言改用 `source_projectile_template_id / target_projectile_template_id` 或现有 projectile 事件字段。
  - `tools/templates/extension_pack_v1` 改为只生成 archetype/mechanic 示例。
  - 当前 wiki、README、AGENTS、模块文档改写为“旧模型已归档，不参与运行时”；历史 ADR 可保留旧术语，但必须标明是历史记录。

## Test Plan

- 静态门禁：
  - `rg "EntityTemplate|TriggerBinding|legacy_template_id|backend_entity_template|entity_template_id|object_template_id|protected_template_id|source_template_id|target_template_id|target_template_ids|legacy_template_id"` 在 `scripts/`、`autoload/`、`data/`、`scenes/`、`tools/templates/` 中必须为 0。
  - `rg "compiled_trigger_bindings|build_runtime_triggers_from_bindings|TriggerBindingRef"` 必须为 0。
  - `rg "template_id"` 只允许出现在 `ProjectileTemplate` 相关命名中。

- 运行验证：
  - 执行 `pwsh "tools/run_all_validations.ps1"`。
  - 重点确认：archetype 实例化、攻击链编译、状态/生命周期、card placement、field object、battle mode objective、extension pack smoke 全部通过。
  - 删除或重写所有 legacy parity/template factory/template instantiation 验证；不允许用“旧路径仍通过”作为成功条件。

- 回归检查：
  - 新建一个临时验证内容，故意引用 `entity_template_id` 或 `legacy_template_id`，应被协议校验拒绝。
  - 新建一个 archetype-only spawn/card/objective 场景，应不依赖任何 template 字段完成生成、命中、死亡和目标判定。

## Assumptions

- `ProjectileTemplate` 不是本次删除对象，但文档中必须明确它是抛射体资源，不是实体旧作者模型。
- 可以破坏所有旧 `.tres` 和旧扩展包兼容；正式内容不提供迁移兜底。
- 旧模型历史允许保留在归档目录和 ADR 历史叙述中，但不得出现在运行时、当前内容入口、扩展模板或验证主线中。
- 不在本计划中执行 git 分支、提交或推送；实施完成后再按用户要求单独处理版本控制。
