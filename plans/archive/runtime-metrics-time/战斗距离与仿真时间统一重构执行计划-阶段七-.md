# 阶段 7 修订计划：从原版源码直算语义数值

## Summary
可以，而且应当改成“原版源码直算”策略。阶段 7 不再用当前 Open PVZ 的 `scan_range/speed/move_speed` 反推 slots，因为这些值可能已经混入历史调参与 world unit 误差。

统一来源优先级：

| 优先级 | 来源 | 用途 |
|---|---|---|
| 1 | `vendor/de-pvz` 原版源码 | 数值唯一优先事实源 |
| 2 | `plans/original-plant-migration-ledger.md` / 原版植物复刻流文档 | 指引应该查哪些原版行为和资源 |
| 3 | 当前 Open PVZ `.tres` | 只用于定位待迁移资源，不作为数值来源 |
| 4 | 当前 Open PVZ 数值 | 仅在源码无法表达且必须兼容时兜底，并记录原因 |

核心公式：

```text
distance_slots = original_px / 80.0
speed_slots_per_sec = original_px_per_tick / 80.0 * 100.0
seconds = original_ticks / 100.0
```

## Key Changes
阶段 7.0：新增源码数值审计门槛。

| 要做什么 | 具体要求 |
|---|---|
| 建立迁移底账 | 新增 `runtime-metrics-time-phase7-original-value-ledger.md`，逐项记录原版源码位置、原版值、换算公式、目标资源、目标字段 |
| 固定换算常量 | `ORIGINAL_SLOT_PX = 80.0`，来自 `Board::GridToPixelX()` / `PixelToGridX()` |
| 固定时间口径 | `ORIGINAL_TICK_HZ = 100.0`，原版 tick 直通 Open PVZ simulation tick |
| 禁止现值反推 | 不使用 `220 / 96`、`55 / 96` 这类当前 world 值反推 |
| 保留证据 | 每个迁移项底账必须写 `vendor/de-pvz/...` 文件和函数名，必要时写源码行附近关键词 |

阶段 7.1：补编译链小修。

| 位置 | 要做什么 |
|---|---|
| `scripts/core/runtime/mechanic_compiler.gd` | `_TARGETING_CONDITION_KEYS` 增加 `&"scan_range_slots"`、`&"range_mode"`，保证 Targeting mechanic 的语义字段能注入 Trigger condition |

阶段 7.2：植物样本按原版源码迁移。

| 类型 | 原版源码位置 | 迁移规则 |
|---|---|---|
| 整行射手 | `Plant::GetPlantAttackRect()` default 分支：`Rect(mX + 60, mY, BOARD_WIDTH, mHeight)` | 使用 `"range_mode": &"full_lane"`，不要写 `scan_range_slots` |
| Peashooter / Snow Pea / Repeater 发射周期 | `gPlantDefs[].mLaunchRate = 150` | 若资源仍未校准，`interval = 1.5` 秒 |
| 直线豌豆速度 | `Projectile::UpdateNormalMotion()`：默认 `mPosX += 3.33f` | `"speed_slots_per_sec": 4.1625` |
| Puff-shroom / Sea-shroom 短射程 | `Plant::GetPlantAttackRect()`：`Rect(mX + 60, mY, 230, mHeight)` | 以原版攻击矩形远端换算，`scan_range_slots = 290 / 80 = 3.625`；底账注明这是矩形到当前标量距离的投影 |
| Fume-shroom 短射程 | `Plant::GetPlantAttackRect()`：`Rect(mX + 60, mY, 340, mHeight)` | `scan_range_slots = 400 / 80 = 5.0`；不从当前 `320.0` 反推 |
| Potato Mine 近身 | `Plant::GetPlantAttackRect()`：`Rect(mX, mY, mWidth - 25, mHeight)`，`mWidth = 80` | `scan_range_slots = 55 / 80 = 0.6875` |
| Tangle Kelp 近身 | `Plant::GetPlantAttackRect()`：`Rect(mX, mY, mWidth, mHeight)`，`mWidth = 80` | `scan_range_slots = 1.0` |
| Hypno-shroom 近身 | 原版行为是被啃咬触发，不是主动圆形扫描 | 本阶段若现有实现仍用 proximity，`scan_range_slots` 用 `55 / 80 = 0.6875` 作为 contact 近似，并在底账标注“协议近似” |

阶段 7.3：投射物 profile 按原版源码迁移。

| 类型 | 原版源码位置 | 迁移规则 |
|---|---|---|
| Pea / Snow Pea / 普通直线投射物 | `Projectile::UpdateNormalMotion()`：`mPosX += 3.33f` | 所有关联 payload/template 使用 `speed_slots_per_sec = 4.1625` |
| Puff projectile | `Projectile::UpdateNormalMotion()` 默认速度 + `MOTION_PUFF` 生命周期 `mProjectileAge >= 75` | 若当前 Puff 使用 projectile，则同样 `4.1625`；若当前用 direct damage，则只迁移射程 |
| Lobbed projectile | `Plant::Fire()`：`mVelX = aRangeX / 120.0f`，`mVelZ = aRangeY / 120.0f - 7.0f`，`mAccZ = 0.115f` | 本阶段不把抛物线速度伪装成固定 `speed_slots_per_sec`；优先保留 `travel_duration = 1.2` 秒或现有 parabola profile 语义，并在底账记录原版 `120 tick` |
| Splash / impact 半径 | `Projectile::IsZombieHitBySplash()`、`DoSplashDamage()`、`KillAllZombiesInRadius()` 等源码 | 只迁移能从源码明确读出的半径；不能从当前 `impact_radius` 反推 |
| Open PVZ profile 字段 | `impact_radius_slots` / `collision_padding_slots` | 仅在有原版源码依据或明确 Open PVZ hitbox 近似说明时填写；否则保留 legacy profile 值并标注“非原版数值，本阶段不校准” |

阶段 7.4：僵尸和割草机也纳入，但不用植物文档背书。

| 类型 | 原版源码位置 | 迁移规则 |
|---|---|---|
| 普通僵尸移动 | `Zombie::PickRandomSpeed()` 默认 `RandRangeFloat(0.23f, 0.32f)`，`Zombie::UpdateZombieWalking()` 消费 `mVelX` | 不用当前 `move_speed = 55.0` 反推；阶段 7 选确定性代表值 `0.275 px/tick`，换算 `move_speed_slots_per_sec = 0.34375` |
| 快速僵尸代表 | `Zombie::PickRandomSpeed()` 中 fast phase/type，如 `0.89f ~ 0.91f` 或对应 Open PVZ archetype 的原版类别 | 若 `archetype_brisk_runner` 映射为“快速跑者”，用 `0.9 px/tick => 1.125 slots/s`；底账必须写明映射假设 |
| 割草机 | `LawnMower::Update()`：草地 mower `aSpeed = 3.33f`，攻击矩形 `Rect(mPosX, mPosY, 50, 80)` | `move_speed_slots_per_sec = 4.1625`；`detection_radius_slots = 50 / 80 = 0.625` |

## 注意事项与不要做
必须注意：

| 规则 | 说明 |
|---|---|
| 原版矩形不是距离 | `Plant::GetPlantAttackRect()` 是矩形；迁移到当前 `scan_range_slots` 时必须在底账写明“矩形投影为标量”的近似 |
| `full_lane` 优先于大数值 | 默认整行射手用 `range_mode = &"full_lane"`，不写 `900`、`4000` 或 `99999` |
| 不美化数值 | 使用公式结果，保留最多 6 位小数；不要为了好看改成 3.0、4.0、0.5 |
| 不用当前值校准原版值 | 当前 `.tres` 只告诉我们改哪里，不告诉我们改成多少 |
| 不扩大协议 | 如果原版行为当前字段表达不了，记录“协议近似”或暂缓，不新增 family |
| 验证失败先查映射 | 不要第一反应改 validation 期望；先查源码值、字段拼写、`range_mode` 注入、单位公式 |

不要做：

| 禁止项 | 原因 |
|---|---|
| 不要从外部 Wiki / 记忆取数 | 项目文档规定 `vendor/de-pvz` 是唯一优先来源 |
| 不要批量替换全仓 `speed` / `scan_range` | 阶段 7 只迁移代表样本 |
| 不要把 lobbed projectile 简化成直线速度 | 原版抛物线是 `120 tick` 轨迹，不是固定横向 speed 表 |
| 不要迁移 showcase 专用 fixture 数值 | showcase 不作为数值校准来源 |
| 不要修改 `vendor/` | 只读参考实现 |
| 不要执行 git commit / push / branch | 用户未要求提交 |

## Assumptions
本阶段采用用户确认的策略：**源码直算优先**，且 **僵尸/割草机纳入阶段 7，但单独从 de-pvz 源码审计，不借原版植物复刻文档背书**。

Open PVZ 当前 `global_position` 作为 gameplay anchor；原版攻击矩形迁移到当前标量距离字段时，使用“从原版 plant origin 到攻击矩形远端”的水平 reach 作为 `scan_range_slots`。如果后续引入矩形 targeting，则这些近似应迁回矩形语义，而不是继续调标量值。

抛物线本阶段不强行归一成 `speed_slots_per_sec`；原版 `120 tick` 轨迹优先映射为 travel duration / parabola profile 语义。
