# 实体活跃性与状态控制计划

> 状态：正式计划
> 更新：2026-05-09
> 适用阶段：前期开发环境，允许一次性破坏旧 API。

## 目标

建立统一的实体活跃性模型，删除分散的旧接口，让“实体自身哪些子系统工作”和“外部世界如何对待这个实体”成为明确、可组合、可验证的运行时语义。

本计划不新增 Mechanic family，不改变 EventBus 模型，不引入完整 HSM。State family 编译器可以输出更丰富的状态 spec；BaseEntity 提供统一 liveness 能力。

## 要解决的问题

当前系统存在以下分散机制：

| 旧机制 | 问题 |
|--------|------|
| `is_combat_active()` | 混合承担目标过滤、可受伤、胜败统计、占位判断等语义 |
| `is_attack_blocked()` | 只能表达攻击暂停，不能表达 movement / trigger / collision 等维度 |
| status `blocks_attack` | 与 controller 执行、状态机、目标过滤割裂 |
| `required_state` | 有价值，但只表达触发器条件，不等于实体活跃性 |
| `FieldObjectRoot.activate()` / `deactivate()` | 场上物件独立发明生命周期开关 |

这些机制会让睡眠、埋地、消化、冻结、割草机待机等行为继续走特判路径。正式方案必须把这些行为收敛到一套多维模型。

## 核心原则

- 不保留兼容层。旧 API 在本计划实施完成后直接删除。
- 不再使用单一 `combat_active` 语义。
- liveness 只控制子系统开关和外部可交互性，不表达队伍、伤害倍率、视觉表现、关卡目标逻辑。
- 状态转换事件只通知下游；转换本身与 liveness 切换必须同步执行。
- `required_state` 保留，它是 Trigger 的业务条件，不是 liveness 的替代品。

## Liveness 维度

### 内省维度

| 维度 | 控制对象 | 典型用途 |
|------|----------|----------|
| `triggers` | TriggerComponent 是否响应 EventBus | 蘑菇睡眠关闭触发器 |
| `state` | StateComponent 是否推进时间转换 | 特殊暂停场景 |
| `movement` | 位移逻辑是否执行 | 冻结、定身 |
| `controllers` | ControllerComponent 是否执行连续策略 | Chomper 消化、冻结攻击 |

### 外视维度

| 维度 | 含义 | 典型用途 |
|------|------|----------|
| `targetable` | Detection / 索敌能否发现它 | Potato Mine 埋地不可选中 |
| `damageable` | HealthComponent 是否接受伤害 | 埋地不可受伤 |
| `collidable` | Projectile / sweep 是否能命中它 | 待机割草机不参与碰撞 |

### 非 liveness 语义

以下语义必须独立命名，不允许继续塞进 targetable/damageable/collidable：

| API | 用途 |
|-----|------|
| `is_runtime_alive()` | 实体是否仍存在于运行时且未进入死亡清理 |
| `is_counted_for_objectives()` | 是否参与胜败、波次目标、模式目标统计 |
| `is_board_occupant()` | 是否参与棋盘占位判断，如需要可由 BoardSlot 查询专用接口表达 |

## 目标 API

BaseEntity 提供：

```gdscript
func is_liveness_enabled(axis: StringName) -> bool
func push_liveness_override(source_id: StringName, profile: Dictionary, priority: int = 20) -> void
func pop_liveness_override(source_id: StringName) -> void

func is_targetable() -> bool
func is_damageable() -> bool
func is_collidable() -> bool
func is_runtime_alive() -> bool
func is_counted_for_objectives() -> bool
```

删除：

```gdscript
func is_combat_active() -> bool
func is_attack_blocked() -> bool
```

FieldObjectRoot 删除：

```gdscript
func activate() -> void
func deactivate() -> void
```

## Override 合成规则

每个 override 包含：

| 字段 | 说明 |
|------|------|
| `source_id` | 来源唯一标识，如 `state:sleeping`、`status:frozen` |
| `profile` | `Dictionary[StringName, bool]`，只写需要覆盖的维度 |
| `priority` | 数字优先级，越大越强 |

合成规则：

- 每个维度单独合成。
- 取优先级最高的 override。
- 相同优先级时 `false` 胜出，采用保守策略。
- 基础 profile 优先级为 0。

默认基础 profile：

| 实体类型 | 默认 profile |
|----------|--------------|
| Plant / Zombie / Projectile | 所有维度 enabled |
| FieldObject | 内省维度按需要 enabled，外视维度默认 disabled |
| Collectible | 不参与 combat liveness，必要时独立处理 |

## StateComponent 集成

### 统一转换入口

StateComponent 必须把时间转换和事件转换收敛到同一个内部入口：

```gdscript
func _execute_transition(transition: Dictionary, reason: Dictionary = {}) -> bool
```

职责：

1. 校验 `from_state` 与当前状态。
2. 校验转换是否已执行。
3. 校验转换条件或外部请求是否合法。
4. 发射 `entity.state_exited`。
5. 同步 pop 旧状态 liveness override。
6. 更新 `current_state`。
7. 同步 push 新状态 liveness override。
8. 同步 owner runtime state。
9. 发射 `entity.state_entered`。

拒绝转换时发射：

```text
entity.state_transition_rejected
```

事件只做通知，不驱动 liveness。

### allowed_transitions 的口径

当前编译出的 `transitions` 已经天然限制 from/to。第一阶段不单独引入重复的 `allowed_transitions` 表。

如果后续需要外部直接请求状态跳转，可增加：

```gdscript
func request_transition(to_state: StringName, reason: Dictionary = {}) -> bool
```

它从已编译 transitions 中寻找当前 `from_state -> to_state` 的合法路径。找不到时发射 `entity.state_transition_rejected`。

### State spec 新字段

State family 编译输出增加：

```gdscript
{
  "initial_state": &"sleeping",
  "transitions": [...],
  "state_liveness": {
    &"sleeping": {
      &"triggers": false,
      &"controllers": false
    }
  }
}
```

内置约定：

| State type | 状态 | liveness |
|------------|------|----------|
| `core.sleeping` | `sleeping` | `triggers=false`, `controllers=false` |
| `core.arming` | `arming` | `targetable=false`, `damageable=false`, `collidable=false` |
| `core.growth` | 默认不改 liveness | 仅表达阶段 |
| `core.rage` | 默认不改 liveness | 仅表达阶段 |

## Status 集成

`apply_status` 删除 `blocks_attack`，保留 `movement_scale`。

新格式：

```gdscript
apply_status(&"frozen", 5.0, {
  "movement_scale": 0.0,
  "liveness_overrides": {
    &"movement": false,
    &"controllers": false
  }
})
```

状态应用时 push：

```text
source_id = status:<status_id>
priority = 20
```

状态过期时 pop 对应 source。

`movement_scale` 与 `movement` liveness 共存：

- `movement=false`：完全不移动。
- `movement=true + movement_scale<1`：减速。

## Effect 集成

第一阶段不新增 core `override_liveness` effect。原因是它会扩大正式 effect 协议面。

如 Coffee Bean 等内容必须直接操作 liveness，优先路径是：

1. 通过状态请求事件触发 StateComponent 合法转换。
2. 必要时由 trusted runtime 扩展 effect 提供直接 push/pop。
3. 若确认需要进入 core effect，再单独走设计审批。

## 子系统迁移表

| 子系统 | 新语义 |
|--------|--------|
| TriggerComponent | `_on_event` 前置检查 `owner.is_liveness_enabled(&"triggers")` |
| BaseEntity.simulation_step | 推进 StateComponent 前检查 `state` 维度 |
| ControllerComponent | 执行前检查 `controllers` 维度 |
| ZombieRoot movement | 移动前检查 `movement`，速度仍乘 `movement_scale` |
| ZombieRoot attack | 删除 `is_attack_blocked()`，controller/attack 由 `controllers` 控制 |
| DetectionRegistry | 目标过滤用 `is_targetable()` |
| HealthComponent | `take_damage` 入口用 `is_damageable()` |
| ProjectileRoot | 碰撞过滤用 `is_collidable()` |
| WaveRunner / BattleModeHost | 目标统计用 `is_counted_for_objectives()` |
| BoardSlot | 占位语义不再使用 combat active，改用明确占位接口或实体 kind/role |

## 实施阶段

### Phase 1：BaseEntity liveness 基础设施

- 新增 liveness override 存储与合成。
- 新增目标 API。
- 删除 `is_combat_active()`、`is_attack_blocked()`。
- 更新 debug snapshot 字段，输出 liveness 结果。

### Phase 2：StateComponent 统一转换入口

- 抽出 `_execute_transition()`。
- 时间转换与事件转换都走统一入口。
- 新增 `entity.state_exited` 和 `entity.state_transition_rejected`。
- 接入 `state_liveness` 的同步 pop/push。

### Phase 3：Status 迁移

- `apply_status` 删除 `blocks_attack`。
- 新增 `liveness_overrides`。
- 过期时 pop status override。

### Phase 4：调用点破坏式替换

- 全仓删除旧 API 引用。
- Detection / Projectile / Zombie / Trigger / Controller / Health / WaveRunner / BattleModeHost / BoardSlot 全部改用明确语义。

### Phase 5：验证与清理

- 新增 liveness 验证场景。
- 运行完整验证矩阵。
- 清理文档中旧 `combat_active` 口径。

## 验收标准

- `rg "is_combat_active|is_attack_blocked|blocks_attack|activate\\(|deactivate\\(" scripts autoload data scenes` 不再出现旧运行时 API 用法。历史文档除外。
- sleeping、arming、frozen、digesting、field_object 待机场景行为可验证。
- `entity.state_entered`、`entity.state_exited`、`entity.state_transition_rejected` 均可被 DebugService / 验证报告观察。
- 现有验证场景全部通过。

## 不做事项

- 不引入完整 HSM。
- 不新增 Mechanic family。
- 不把视觉、音频表现写入 liveness。
- 不在第一阶段新增 core `override_liveness` effect。
