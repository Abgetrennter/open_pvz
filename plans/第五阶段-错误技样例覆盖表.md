# 第五阶段-错误技样例覆盖表

> 这份覆盖表用于回答：当前第五阶段样例已经覆盖了哪些能力域，哪些适合作为 smoke test，哪些适合作为长期回归基线。

| 场景 ID | 样例类型 | 覆盖能力域 | 关键观察点 | Smoke Test | 回归基线 |
|------|------|------|------|------|------|
| `reactive_retaliation_chaos_validation` | 受伤反击型 | `when_damaged`、事件回写、双来源反击 | 两个攻击者都能触发反击，反击仍指向正确事件源 | 是 | 是 |
| `death_blossom_chaos_validation` | 死亡爆炸型 | `on_death`、`explode`、多目标范围伤害 | 死亡链能稳定引出多目标爆炸伤害 | 是 | 是 |
| `tracking_barrage_chaos_validation` | 追踪抛射型 | 追踪投射、空地并行、高度差异 | 地面/空中追踪样例共存且各自命中正确目标 | 是 | 是 |
| `chain_explosion_cascade_validation` | 级联爆炸型 | 多源反击 + 死亡爆炸链 | 两种投射类型攻击同一 bomber，bomber 分别反击两个事件源，死亡后爆炸伤害墙壁 | 是 | 是 |
| `splash_zone_cascade_validation` | 溅射打击型 | 终端 `on_hit` 溅射、跨类型目标 | 同一爆炸同时伤害 tank 和 walker | 是 | 是 |
| `fast_pursuit_cascade_validation` | 高速追踪型 | 追踪高速目标、伤害积累 | 追踪投射物可靠命中 80px/s 移动目标 | 否 | 是 |
| `multi_lane_retaliation_cascade_validation` | 多车道反击型 | 多车道反击隔离、事件源解析 | 两条车道反击不串道，各自指向正确事件源 | 否 | 是 |
| `extension_pack_smoke_validation` | 扩展入口 smoke test | 扩展扫描、扩展模板注册、扩展场景加载 | 外部模板和外部场景可被加载并进入统一验证链 | 是 | 否 |
| `hit_split_chaos_validation` | 命中分裂型 | 扩展 `on_hit` effect、分裂追击、命中点二次生成 | 主弹先命中前排目标，再从命中点分裂出 follow-up 投射物 | 是 | 是 |
| `periodic_summon_chaos_validation` | 周期召唤型 | 扩展 `spawn_entity` effect、共享 spawn 主链、召唤单位参战 | summoner 会周期生成 shardling，召唤体继续进入统一攻击链 | 是 | 是 |
| `apply_status_chaos_validation` | 状态控制型 | 扩展 `apply_status` effect、共享状态系统 | 命中后直接挂减速状态，并触发现有 `entity.status_applied / status.effect_observed` 观察点 | 是 | 是 |
| `knockback_chaos_validation` | 击退控制型 | 扩展 `knockback` effect、最小位移回写 | 命中后发出 `entity.knockback_applied`，并把目标向后推回 | 是 | 是 |
| `chain_bounce_chaos_validation` | 跳链伤害型 | 扩展 `chain_bounce` effect、带衰减多目标伤害 | 伤害从首目标跳到邻近目标，并以不同伤害值体现在事件流里 | 是 | 是 |
| `aura_chaos_validation` | 光环控制型 | 扩展 `aura` effect、持续范围控制、共享状态系统 | 站场植物周期减速近处敌人，并保持远处敌人不受影响 | 是 | 是 |
| `delayed_trigger_chaos_validation` | 延迟触发型 | 扩展 `delayed_trigger` effect、时序延迟单体结算 | 命中后不立刻伤害，而是在延迟后才触发单体伤害 | 是 | 是 |
| `delayed_explode_chaos_validation` | 延迟爆炸型 | 扩展 `delayed_explode` effect、时序型范围爆炸 | 命中后延迟一段时间再引爆并波及多个目标 | 是 | 是 |
| `mark_chaos_validation` | 标记型 | 扩展 `mark` effect、独立标记生命周期 | 命中后施加 mark，并在持续时间结束后触发 mark_removed | 是 | 是 |
